## 3.1 Hidden Regime Process

### The Idea
First conceptual step in a Markov Switching Model is to separate the  world into two layers

| Layer             | Symbol | Observed | Meaning                                      |
| ----------------- | ------ | -------- | -------------------------------------------- |
| Observation layer | $y_t$  | Yes      | Actual Time Series Value                     |
| Regime Layer      | $S_t$  | No       | The hidden state/regime that generated $y_t$ |
So model assumes that the observed time series is not generated by one fixed statistical "mechanism" or a distribution. Instead it is generated by several hidden regimes.

For example, suppose $y_t$ is a financial return, commodity return, log-price difference, volatility proxy, or some transformed market feature. The model assumes that, at each time $t$, the market is in some hidden condition:

$$S_t\in\set{1,\dots,K}$$
the value of $S_t$ affects how $y_t$ is generated. This is the central latent-variable idea.

### Observed Sequence

Let the observed time series be:

$$y_{1:T}=(y_1,\dots,y_T)$$
Let this notation mean a sample od the timeseries in a time window from $1$ to $T$.

### Hidden Regime Sequence

The model assumes that behind the observed sequence there is an latent regime sequence:
$$S_{1:T} = (S_1,\dots,S_T)$$
Each unobserved state in the sequence is:
$$S_t\in\set{1,\dots,K}$$
Where:
 - $K$ is the number of regimes
 - $S_t=n$ the process is in regime $n$ at time $t$ where $n\in[k]$ 

For Example:
With $K = 2$, $S_t\in\set{1,2}$.
Possible Interpretation:

| Hidden state | Interpretation                     |
| -----------: | ---------------------------------- |
|      $S_t=1$ | calm / low-volatility regime       |
|      $S_t=2$ | turbulent / high-volatility regime |
These kinds of interpretations can be inferred after fitting the model. The model only knows the regimes as labels ($1,\dots,K$).
The economic, financial, or statistical meaning is assigned by inspecting the fitted parameters and posterior probabilities.

### Regimes are latent, not observed

The "real" regime sequence ($S_1,S_2,\dots,S_T$) is unknown. Therefore the model cannot say wit certainty $S_t=1$ instead it estimates the probabilities:
$$\Pr(S_t=j | y_{1:T})$$

That means:

> Given all observed data, what is the probability that time $t$ belonged to regime $j$?

This is why Markov switching models are probabilistic regime-detection models, not deterministic labeling algorithms

### Why a hidden process

A standard single-regime model might assume:

$$y_t \sim \mathcal{N}(\mu,\sigma^2)$$

for all $t$.

That means the whole time series is described using one mean and one variance.

But many real time series are not homogeneous. They may have periods where:

- volatility is low;
- volatility is high;
- the mean changes;
- the distribution becomes more unstable;
- extreme values become more likely;
- persistence changes.

A Markov switching model handles this by saying:

$$y_t \mid S_t=j$$

has a distribution depending on $j$.

In your Gaussian model:
$$y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)$$
So instead of one global Gaussian, the model has $K$ possible Gaussian regimes.

### Core generative idea
The hidden regime process is part of a generative story.

The model imagines that the data were generated as follows:

1. At the first time point, choose an initial regime:

$$S_1 \sim \pi$$

2. For each later time point, choose the next regime using the Markov transition matrix:

$$S_t \mid S_{t-1} \sim A_{S_{t-1},\cdot}$$

3. Generate the observation from the regime-specific distribution:

$$y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)$$

So there are two linked processes:
```
hidden regime process:
S_1 → S_2 → S_3 → ... → S_T

observed process:
y_1   y_2   y_3   ...   y_T
```

More explicitly:
```
S_1 → S_2 → S_3 → ... → S_T
↓     ↓     ↓           ↓
y_1   y_2   y_3         y_T
```
The arrows mean:

- $S_t$ depends on $S_{t-1}$;
- $y_t$ depends on $S_t$.

This is why the model is closely related to a Hidden Markov Model, but in econometrics and time-series analysis this specification is usually called a **Markov switching model**. Hamilton’s work is the classical source for this formulation in time-series econometrics \cite{Hamilton1989,Hamilton1994}

### Mathematical object: the hidden state space

The hidden state space is finite:

$$\mathcal{S} = \{1,\dots,K\}$$

The hidden regime variable is therefore a discrete random variable:

$$S_t : \Omega \to \mathcal{S}$$

For every $j \in {1,\dots,K}$:

$$\Pr(S_t=j)$$

is the probability that the process is in regime $j$ at time $t$.

But in inference we usually condition on observations, so the important quantities are posterior probabilities:

$$\Pr(S_t=j \mid y_{1:t})$$

or:

$$\Pr(S_t=j \mid y_{1:T})$$

These are different.

| Quantity                  | Meaning                                             |
| ------------------------- | --------------------------------------------------- |
| $\Pr(S_t=j)$              | unconditional probability of regime $j$ at time $t$ |
| $\Pr(S_t=j \mid y_{1:t})$ | filtered probability using data up to time $t$      |
| $\Pr(S_t=j \mid y_{1:T})$ | smoothed probability using the whole sample         |

This distinction becomes essential later in filtering and smoothing.

### Randomness

In the model, both $S_t$ and $y_t$ are random variables.

The hidden state $S_t$ is random because the model does not deterministically know which regime applies.

The observation $y_t$ is random because even within a fixed regime, observations vary according to the regime-specific distribution.

So for every time $t$:
$$S_t \sim \text{categorical distribution over } \{1,\dots,K\}$$

and then:
$$y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)$$

The uncertainty has two levels:

|Source of uncertainty|Meaning|
|---|---|
|Regime uncertainty|Which state generated the observation?|
|Observation uncertainty|Given the state, what value was generated?|

This layered uncertainty is exactly why the model is useful.

It can distinguish between:

- an observation that is extreme but still plausible in a high-volatility regime;
- an observation that is unlikely under all regimes;
- an observation that causes posterior mass to shift from one regime to another.

### Posterior Regime Probabilities
Because $S_t$ is hidden, the most important output is not a regime label but a posterior distribution:
$$\left( \Pr(S_t=1 \mid y_{1:T}), \dots, \Pr(S_t=K \mid y_{1:T}) \right)$$

For example, with $K=3$, at time $t$ the model may output:
$$(0.05, 0.10, 0.85)$$

This means:

|Regime|Posterior probability|
|---|---|
|1|$0.05$|
|2|$0.10$|
|3|$0.85$|

So the model strongly favors regime 3, but it still represents uncertainty.

A deterministic regime label could be obtained by:
$$\hat{S}_t = \arg\max_j \Pr(S_t=j \mid y_{1:T})$$

In the example:
$$\hat{S}_t = 3$$

But this hard label throws away information.

The probability vector:
$$(0.05,0.10,0.85)$$

is more informative than just saying:
$$\hat{S}_t=3$$

In your thesis, it is safer to emphasize posterior probabilities rather than hard regime labels.

### Filtering versus smoothing (prolog)
Even though filtering and smoothing will be discussed later, you should foreshadow them here.
There are two main kinds of posterior regime probabilities:
#### Filtered probabilities

$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$

These use only information available up to time $t$.
They answer:
> At time $t$, after seeing observations up to $t$, what is the probability of being in regime $j$?

These are useful for online detection.
#### Smoothed probabilities

$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$

These use the full sample.
They answer:

> After observing the whole time series, what is the probability that time $t$ belonged to regime $j$?

These are useful for offline analysis and EM estimation.
The distinction matters because future observations can change how we interpret past observations.

Example:
- At time $t$, one observation is ambiguous.
- At times $t+1,t+2,t+3$, the series strongly resembles a high-volatility regime.
- The smoother may revise the probability that time $t$ was already part of the high-volatility regime.

So:
$$\Pr(S_t=j \mid y_{1:t})$$

and:
$$\Pr(S_t=j \mid y_{1:T})$$

need not be equal.

### Switching

The word “switching” means that the data-generating mechanism can change over time.
For example, suppose $K=2$ and:
$$S_t = \begin{cases} 1, & \text{low-volatility regime},\\ 2, & \text{high-volatility regime}. \end{cases}$$

Then the observation model is:
$$y_t \mid S_t=1 \sim \mathcal{N}(\mu_1,\sigma_1^2)$$
$$y_t \mid S_t=2 \sim \mathcal{N}(\mu_2,\sigma_2^2)$$

If the hidden process is:
$$S_{1:8} = (1,1,1,2,2,2,1,1)$$

then the time series is generated by regime 1 for the first three observations, regime 2 for the next three, and regime 1 again for the final two.
So the model switches between parameter sets:
$$(\mu_1,\sigma_1^2)$$

and:
$$(\mu_2,\sigma_2^2)$$
This is the core idea of regime switching.

## 3.2 Markov chain dynamics

### Intro

In section 3.1 we introduced a latent regime sequence
$$S_{1:T}=(S_1,\dots,S_T)$$

where:
$$S_t\in\set{1,\dots,K}$$
In section 3.2 we establish how this hidden regime sequence behaves over time.

The key assumption is:

> The hidden regimes evolve according to a first-order Markov chain.

That means the next regime depends on the current regime, but not on the full past history once the current regime is known.
Formally:
$$\Pr(S_t=j \mid S_{t-1}=i, S_{t-2},\dots,S_1) = \Pr(S_t=j \mid S_{t-1}=i)$$
So the entire past regime history is summarized by the most recent regime $S_{t-1}$.
This is the **Markov property**.

### Initial regime distribution
The hidden process must begin somewhere. The first state is drawn from an initial distribution:
$$S_1 \sim \pi$$
The vector:
$$\pi = (\pi_1,\dots,\pi_K)$$
contains the initial probabilities:
$$\pi_j = \Pr(S_1=j)$$
Because $\pi$ is a probability vector, it must satisfy:
$$\pi_j \ge 0$$
and:
$$\sum_{j=1}^K \pi_j = 1$$
#### Intuition
The vector $\pi$ describes what the model believes about the initial regime before observing the first data point.

For example, with $K=2$:
$$\pi = (0.8,0.2)$$
means:
$$\Pr(S_1=1)=0.8$$
$$\Pr(S_1=2)=0.2$$
So before seeing $y_1$, the model believes regime 1 is more likely.
In practice, $\pi$ often matters most near the beginning of the sample. For long time series, the effect of the initial distribution usually becomes less important than the transition matrix and emission distributions.

### Transition probabilities

For $t \ge 2$, the hidden state evolves according to the transition rule:
$$\Pr(S_t=j \mid S_{t-1}=i)=A_{ij}$$
Here:
- $i$ is the previous regime;
- $j$ is the next regime;
- $A_{ij}$ is the probability of moving from regime $i$ to regime $j$.

The transition probabilities are collected in the transition matrix:
$$A = \begin{pmatrix} A_{11} & A_{12} & \cdots & A_{1K} \\ A_{21} & A_{22} & \cdots & A_{2K} \\ \vdots & \vdots & \ddots & \vdots \\ A_{K1} & A_{K2} & \cdots & A_{KK} \end{pmatrix}$$
Each row corresponds to a current regime. Each column corresponds to the next regime.
So row $i$ is:
$$A_{i,\cdot} = (A_{i1},A_{i2},\dots,A_{iK})$$
This row is the distribution of the next state given that the current state is $i$:
$$S_t \mid S_{t-1}=i \sim \text{Categorical}(A_{i1},\dots,A_{iK})$$

### Stochastic matrix constraints
Because each row of $A$ is a probability distribution, it must satisfy:
$$A_{ij}\ge0$$
for every $i,j$, and:
$$\sum_{j=1}^K A_{ij}=1$$
for every row $i$.
A matrix with nonnegative rows summing to one is called a **row-stochastic matrix**.
So in this model:
$$A \in [0,1]^{K \times K}$$
and:
$$A\mathbf{1} = \mathbf{1}$$
where $\mathbf{1}$ is a vector of ones.

#### Intuition
If the process is currently in regime $i$, then it must transition to **some** regime at the next time step.
That includes the possibility of staying in the same regime.
So the probabilities:
$$A_{i1}, A_{i2}, \dots, A_{iK}$$
must add up to one.

### Diagonal entries and persistence
The diagonal element:
$$A_{jj}$$
is the probability that the process stays in regime $j$ for one more time step:
$$A_{jj} = \Pr(S_t=j \mid S_{t-1}=j)$$
Therefore:
- high $A_{jj}$ means high persistence;
- low $A_{jj}$ means low persistence.
### Off Diagonal Entries and Switching
The off-diagonal element:
$$A_{ij}, \quad i \ne j$$
is the probability of switching from regime $i$ to regime $j$:
$$A_{ij} = \Pr(S_t=j \mid S_{t-1}=i)$$
Large off-diagonal entries mean frequent transitions.
Small off-diagonal entries mean rare transitions.
For $K=2$:
$$A = \begin{pmatrix} 0.95 & 0.05 \\ 0.10 & 0.90 \end{pmatrix}$$
the switching probabilities are:
$$A_{12}=0.05$$
$$A_{21}=0.10$$
So switching from regime 2 to regime 1 is twice as likely as switching from regime 1 to regime 2.
That asymmetry can be meaningful.
For example, in financial data, transitions into stress regimes may be rare but abrupt, while transitions back to calm regimes may have different dynamics.

### Expected regime duration
A very useful interpretation of the transition matrix is expected duration.
Suppose the process is currently in regime $j$.
At each next time step, it stays in regime $j$ with probability:
$$A_{jj}$$
It leaves regime $j$ with probability:
$$1-A_{jj}$$
Therefore, the duration $D_j$ of a spell in regime $j$ follows a geometric distribution:
$$\Pr(D_j=d) = (A_{jj})^{d-1}(1-A_{jj}), \qquad d=1,2,\dots$$
The expected duration is:
$$\mathbb{E}[D_j] = \frac{1}{1-A_{jj}}$$
#### Example
If:
$$A_{jj}=0.95$$
then:
$$\mathbb{E}[D_j] = \frac{1}{1-0.95} = \frac{1}{0.05} = 20$$
So regime $j$ lasts 20 time steps on average.
If the data are daily, this means about 20 trading days.
If the data are monthly, this means about 20 months.
#### Intuition
This formula gives a very interpretable summary of persistence.
Instead of saying:
$$A_{jj}=0.95$$
you can say: "The fitted model implies that regime $j$ lasts approximately 20 observations on average"

### Important limitation: geometric durations
The expected duration formula also reveals a limitation.
Because the process is Markovian, the probability of leaving regime $j$ is always:
$$1-A_{jj}$$
regardless of how long the process has already been in that regime.
That means the model has **memoryless regime durations**.
For example, if:
$$A_{jj}=0.95$$
then the probability of leaving is always:
$$0.05$$
It is $0.05$ after one time step, and still $0.05$ after twenty time steps.
This is a direct consequence of the Markov assumption.
#### Why this matters
This is useful to mention briefly because it motivates possible extensions such as hidden semi-Markov models, where duration distributions are modeled explicitly.
But in your current model, ordinary Markov dynamics are the correct foundation.
### Joint probability of a hidden regime path
The transition matrix gives the probability of an entire hidden path.
For a state sequence:
$$s_{1:T} = (s_1,\dots,s_T)$$
the probability is:
$$\Pr(S_{1:T}=s_{1:T}) = \pi_{s_1} \prod_{t=2}^T A_{s_{t-1},s_t}$$
This formula is important because it later appears inside the complete-data likelihood:
$$p(y_{1:T},S_{1:T}\mid\theta) = \pi_{S_1} \prod_{t=2}^T A_{S_{t-1},S_t} \prod_{t=1}^T f_{S_t}(y_t)$$
#### Intuition
A path is likely if:
1. it starts in a regime with high initial probability;
2. it uses transitions with high probability;
3. it stays in persistent regimes when diagonal probabilities are high.
For example, if $A_{11}=0.98$, then a path that remains in regime 1 for many periods may have relatively high transition probability.
If $A_{12}=0.01$, then a path that frequently switches from 1 to 2 is unlikely.

#### Note Seed
The hidden regime sequence is modeled as a finite-state first-order Markov
chain. Thus, conditional on the current regime, the distribution of the next
regime does not depend on the earlier regime history:
$$
\Pr(S_t=j \mid S_{t-1}=i,S_{t-2},\dots,S_1)
=
\Pr(S_t=j \mid S_{t-1}=i).
$$
The initial state is distributed according to
$$
S_1 \sim \pi,
$$
where $\pi_j = \Pr(S_1=j)$, $\pi_j \ge 0$, and
$$
\sum_{j=1}^K \pi_j = 1.
$$
For $t \ge 2$, the transition probabilities are collected in a
row-stochastic matrix $A$, whose entries are defined by
$$
A_{ij} = \Pr(S_t=j \mid S_{t-1}=i).
$$
Each row of $A$ is a probability distribution over the next regime, so
$$
A_{ij} \ge 0,
\qquad
\sum_{j=1}^K A_{ij}=1.
$$

The transition matrix determines the persistence and switching behavior of the
latent regime process. The diagonal entry $A_{jj}$ gives the probability of
remaining in regime $j$ for one more period, while off-diagonal entries
$A_{ij}$, $i \ne j$, represent switching probabilities. Under the ordinary
Markov specification, regime durations are geometrically distributed. Therefore,
the expected duration of regime $j$ is
$$
\mathbb{E}[D_j] = \frac{1}{1-A_{jj}}.
$$
This quantity provides a direct interpretation of fitted diagonal transition
probabilities as average regime persistence.
The transition matrix also plays a central role in inference. In the Hamilton
filter, it propagates filtered probabilities forward through the prediction
equation
$$
\alpha_{t|t-1}(j)
=
\sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i).
$$
Thus, the posterior probability of a regime at time $t$ depends not only on
the observation at that time, but also on the probability of transitioning from
previous regimes. This temporal dependence distinguishes Markov switching
models from static mixture models and gives the inferred regime probabilities a
natural persistence structure \cite{Hamilton1989,Hamilton1994}.

#### Key summary
The core of Section 3.2 is:
$$S_1 \sim \pi$$
$$\Pr(S_t=j \mid S_{t-1}=i)=A_{ij}$$
$$\sum_j A_{ij}=1, \qquad A_{ij}\ge 0$$

The transition matrix controls:
- persistence;
- switching;
- expected regime duration;
- temporal smoothness of inferred regimes;
- the prediction step in filtering;
- expected transition counts in EM.

## 3.3 Regime-dependent Gaussian observation model
### What this section is trying to establish
Sections 3.1 and 3.2 define the **hidden regime process**:
$$S_t \in \{1,\dots,K\}$$
with regime dynamics governed by an initial distribution $\pi$ and transition matrix $A$.
Section 3.3 defines the second layer of the model:
> Given that the process is in regime $j$ at time $t$, how is the observed value $y_t$ generated?

This is called the **observation model**, **emission model**, or **regime-dependent conditional distribution**.

### Observations depend on hidden regimes
The model assumes that the distribution of $y_t$ depends on the hidden state $S_t$.
Formally:
$$y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)$$
This means:

> If the hidden regime at time $t$ is $j$, then the observation $y_t$ is drawn from a Gaussian distribution with regime-specific mean $\mu_j$ and regime-specific variance $\sigma_j^2$.

So each regime has its own Gaussian parameters:
$$(\mu_j,\sigma_j^2)$$
For $K$ regimes, the observation model has:
$$(\mu_1,\sigma_1^2),\dots,(\mu_K,\sigma_K^2)$$
The full emission parameter set is:
$$\{(\mu_j,\sigma_j^2)\}_{j=1}^K$$
This is the part of the Markov switching model that makes regimes statistically different from each other.
Hamilton’s Markov-switching framework uses regime-dependent observation equations; your model specializes this to Gaussian regime-conditional distributions \cite{Hamilton1989,Hamilton1994}.
### What does $\sigma_j^2$ mean?
The parameter $\sigma_j^2$ is the **conditional variance** of the observation in regime $j$:
$$\sigma_j^2 = \operatorname{Var}(y_t \mid S_t=j)$$
The standard deviation is:
$$\sigma_j = \sqrt{\sigma_j^2}$$
Variance controls the spread of the observations around the regime mean.
A small variance means observations in that regime are tightly concentrated around $\mu_j$.
A large variance means observations in that regime are more dispersed.
#### Financial intuition
In financial or commodity-market applications, $\sigma_j^2$ is often more interpretable than $\mu_j$.
For example:

| Regime | Mean      | Variance | Possible interpretation            |
| ------ | --------- | -------- | ---------------------------------- |
| $j=1$  | near zero | small    | calm / low-volatility regime       |
| $j=2$  | near zero | large    | turbulent / high-volatility regime |

So even if two regimes have similar means, they may still be distinct because their variances differ.
This is especially important for returns, where means are often small and noisy, while volatility differences are more visible.

### The Gaussian density
The conditional density of $y_t$ in regime $j$ is:
$$f_j(y_t) = p(y_t \mid S_t=j;\theta)$$
For the Gaussian model:
$$f_j(y_t) = \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right)$$

This function assigns a likelihood value to the observation $y_t$ under regime $j$.
#### What this density does
The density is large when $y_t$ is close to $\mu_j$.
The density becomes smaller as $y_t$ moves farther away from $\mu_j$.
The rate at which it becomes smaller depends on $\sigma_j^2$.
If $\sigma_j^2$ is small, the density penalizes deviations from $\mu_j$ strongly.
If $\sigma_j^2$ is large, the density is more tolerant of large deviations.

###  Same observation, different regime likelihoods
The same observation $y_t$ is evaluated under every regime.
For each regime $j$, the model computes:
$$f_j(y_t)$$
So at time $t$, the model asks:
$$f_1(y_t), f_2(y_t), \dots, f_K(y_t)$$
Each value answers:
> How plausible is observation $y_t$ if regime $j$ were active?
#### Example
Suppose $K=2$ and:
$$\mu_1 = 0, \qquad \sigma_1^2 = 1$$
$$\mu_2 = 0, \qquad \sigma_2^2 = 9$$
Both regimes have the same mean, but regime 2 has much larger variance.
Now suppose:
$$y_t = 4$$
Under regime 1:
$$y_t=4$$
is four standard deviations away from the mean.
Under regime 2:
$$y_t=4$$
is only:
$$\frac{4}{3}$$
standard deviations away from the mean.
So the observation is much more plausible under regime 2.
That does not prove $S_t=2$, but it increases the posterior probability of regime 2.
###  Why the model uses log-density
The Gaussian density can become very small, especially when:
- $|y_t-\mu_j|$ is large;
- $\sigma_j^2$ is small;
- the time series is long and likelihoods are multiplied across many observations.
Directly multiplying small probabilities can cause numerical underflow.
So the implementation computes the log-density:
$$\log f_j(y_t)$$
For the Gaussian density:
$$\log f_j(y_t) = -\frac{1}{2}\log(2\pi) -\frac{1}{2}\log(\sigma_j^2) - \frac{(y_t-\mu_j)^2}{2\sigma_j^2}$$

### Deriving the log-density
Start from the Gaussian density:
$$f_j(y_t) = \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right)$$
Take logarithms:
$$\log f_j(y_t) = \log \left[ \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right) \right]$$
Use:
$$\log(ab)=\log a+\log b$$
so:
$$\log f_j(y_t) = \log \left( \frac{1}{\sqrt{2\pi\sigma_j^2}} \right) + \log \left[ \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right) \right]$$
The second term simplifies:
$$\log(e^x)=x.$$
Therefore:
$$\log \left[ \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right) \right] = -\frac{(y_t-\mu_j)^2}{2\sigma_j^2}$$
For the first term:
$$\log \left( \frac{1}{\sqrt{2\pi\sigma_j^2}} \right) = -\log \left( \sqrt{2\pi\sigma_j^2} \right)$$
Since:
$$\sqrt{2\pi\sigma_j^2} = (2\pi\sigma_j^2)^{1/2}$$
we get:
$$-\log \left( (2\pi\sigma_j^2)^{1/2} \right) = -\frac{1}{2}\log(2\pi\sigma_j^2)$$
Then:
$$-\frac{1}{2}\log(2\pi\sigma_j^2) = -\frac{1}{2}\log(2\pi) - \frac{1}{2}\log(\sigma_j^2)$$
Thus:
$$\log f_j(y_t) = -\frac{1}{2}\log(2\pi) -\frac{1}{2}\log(\sigma_j^2) - \frac{(y_t-\mu_j)^2}{2\sigma_j^2}$$
This is the form used in the model because it separates the constant term, the variance penalty, and the squared-error penalty.

### Interpreting each term in the log-density
The log-density is:
$$\log f_j(y_t) = -\frac{1}{2}\log(2\pi) -\frac{1}{2}\log(\sigma_j^2) - \frac{(y_t-\mu_j)^2}{2\sigma_j^2}$$
It has three parts.
#### Constant term
$$-\frac{1}{2}\log(2\pi)$$
This is the same for all regimes and all observations.
It is necessary for the correct Gaussian density, but it does not by itself distinguish regimes.
#### Variance normalization term
$$-\frac{1}{2}\log(\sigma_j^2)$$
This term penalizes larger variances.
A Gaussian with a large variance spreads probability mass over a wider region. Because it spreads mass more widely, its peak density is lower.
So large-variance regimes are more tolerant of extreme observations, but they pay a normalization cost.
##### Intuition
A high-volatility regime can explain extreme observations better.
But it cannot assign high density to every ordinary observation for free, because its distribution is more spread out.
#### Squared-error term
$$-\frac{(y_t-\mu_j)^2}{2\sigma_j^2}$$
This term penalizes the distance between the observation and the regime mean.
The penalty is based on squared distance:
$$(y_t-\mu_j)^2$$
But the penalty is scaled by the regime variance:
$$\sigma_j^2$$
So the same deviation is less surprising in a high-variance regime.
##### Example
Suppose:
$$y_t-\mu_j = 2$$
If:
$$\sigma_j^2=1$$
then the squared-error penalty is:
$$\frac{2^2}{2\cdot 1} = -2$$
If:
$$\sigma_j^2=4$$
then:
$$-\frac{2^2}{2\cdot 4} = -\frac{1}{2}$$
The same deviation is penalized less strongly under the higher-variance regime.
### Emissions do not determine regimes alone
A common misunderstanding is:
> The regime with the largest emission density is the selected regime.

That is not quite correct.
The emission density is only one component.
Filtering combines:
1. the prior predicted regime probability;
2. the emission likelihood.
The update equation later is:
$$\alpha_{t|t}(j) = \frac{ f_j(y_t)\alpha_{t|t-1}(j) }{ \sum_{\ell=1}^K f_\ell(y_t)\alpha_{t|t-1}(\ell) }$$
So the posterior probability of regime $j$ is proportional to:
$$f_j(y_t)\alpha_{t|t-1}(j)$$
##### Intuition
A regime becomes likely if:
- it was already plausible before seeing $y_t$;
- and it explains $y_t$ well.
A high emission density can be outweighed by a very low predicted regime probability.
This is the difference between Markov switching and independent clustering.

### Conditional independence assumption

The observation model includes an important conditional independence assumption.
Given the hidden state $S_t$, the current observation $y_t$ is generated from the regime-specific Gaussian distribution.
In the simplest version used here, this means:
$$p(y_t \mid S_t, S_{t-1},\dots,S_1, y_{1:t-1}) = p(y_t \mid S_t)$$
And with the Gaussian specification:
$$p(y_t \mid S_t=j) = f_j(y_t)$$

### Complete observation likelihood conditional on a hidden path
Suppose the entire hidden path were known:
$$S_{1:T}=s_{1:T}$$
Then the conditional density of observations is:
$$p(y_{1:T} \mid S_{1:T}=s_{1:T},\theta) = \prod_{t=1}^T f_{s_t}(y_t)$$
Because:
$$y_t \mid S_t=s_t \sim \mathcal{N}(\mu_{s_t},\sigma_{s_t}^2)$$
So:
$$p(y_{1:T} \mid S_{1:T}=s_{1:T},\theta) = \prod_{t=1}^T \frac{1}{\sqrt{2\pi\sigma_{s_t}^2}} \exp \left( -\frac{(y_t-\mu_{s_t})^2}{2\sigma_{s_t}^2} \right)$$
The log version is:
$$\log p(y_{1:T} \mid S_{1:T}=s_{1:T},\theta) = \sum_{t=1}^T \log f_{s_t}(y_t)$$
#### Why this matters
This is the observation part of the complete-data likelihood.
Later, EM replaces the unknown regime indicators with posterior probabilities.
But the Gaussian emission model is the piece that determines how observations contribute to estimating $\mu_j$ and $\sigma_j^2$.
#### Indicator-variable view
It is often useful to write the emission likelihood using indicators.
Define:
$$\mathbf{1}\{S_t=j\} = \begin{cases} 1, & S_t=j,\\ 0, & S_t\ne j. \end{cases}$$
Then the conditional observation log-likelihood can be written as:
$$\sum_{t=1}^T \sum_{j=1}^K \mathbf{1}\{S_t=j\} \log f_j(y_t)$$
This says:
> At each time $t$, only the active regime contributes to the complete-data log-likelihood.

If $S_t=2$, then:
$$\mathbf{1}\{S_t=2\}=1$$
and all other regime indicators are zero.
### Limitation of Gaussian emissions
The Gaussian assumption is convenient, but it has limitations.
A Gaussian distribution has relatively light tails.
So if the data contain extreme observations, jumps, or heavy-tailed behavior, the Gaussian model may interpret them as strong evidence for a high-variance regime or may distort variance estimates.
This is why a natural extension is a Student-$t$ emission model:
$$y_t \mid S_t=j \sim t_{\nu_j}(\mu_j,\sigma_j^2)$$
The Student-$t$ distribution has heavier tails, so it is more robust to extreme observations.
Do not develop this fully in this section as it is not implemented. A short note is enough:
> A possible extension is to replace Gaussian emissions with Student-$t$ emissions to improve robustness to heavy-tailed observations.

That gives you a clean future-work or extension bridge without distracting from the central Gaussian model.
### Summary
The observation layer of the model specifies how the observed value $y_t$ is
generated conditional on the latent regime $S_t$. In the Gaussian
Markov-switching specification used in this work, each regime $j$ is associated
with its own conditional mean $\mu_j$ and variance $\sigma_j^2$. Thus,
$$
y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2).
$$
The corresponding conditional density is denoted by
$$
f_j(y_t) = p(y_t \mid S_t=j;\theta),
$$
and is given by
$$
f_j(y_t)
=
\frac{1}{\sqrt{2\pi\sigma_j^2}}
\exp\left(
-\frac{(y_t-\mu_j)^2}{2\sigma_j^2}
\right).
$$
Equivalently, the log-density is
$$
\log f_j(y_t)
=
-\frac{1}{2}\log(2\pi)
-\frac{1}{2}\log(\sigma_j^2)
-
\frac{(y_t-\mu_j)^2}{2\sigma_j^2}.
$$

The density $f_j(y_t)$ measures how plausible the observation $y_t$ is under
regime $j$. Therefore, the same observation is evaluated under all regimes.
A regime with a mean close to $y_t$ and a suitable variance assigns higher
density to that observation, while regimes for which $y_t$ is far from the
conditional mean assign lower density. In financial applications, regimes are
often distinguished primarily by their fitted variances, with low-variance
regimes interpreted as calmer periods and high-variance regimes interpreted as
turbulent periods.

The emission density alone does not determine the regime. In the filtering
recursion, it is combined with the one-step-ahead predicted regime probability.
Thus, the posterior probability of regime $j$ depends both on how likely the
regime was before observing $y_t$ and on how well its Gaussian density explains
$y_t$. This is what distinguishes the model from an independent Gaussian
mixture model.
#### Key conceptual summary
The section is built around this equation:
$$y_t \mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)$$
It means:
> Each hidden regime defines a different Gaussian explanation for the observed value.

The Gaussian density:
$$f_j(y_t)$$
measures how plausible $y_t$ is under regime $j$.
The filter later combines:
$$f_j(y_t)$$
with the predicted regime probability:
$$\alpha_{t|t-1}(j)$$
So the emission model supplies the observation evidence; the Markov chain supplies temporal persistence.

## 3.4 Observed-data likelihood
### What this section should do
This section should answer one central question:
> Given parameters $\theta$, how likely is the observed time series $y_{1:T}$ under the Markov switching model, when the regime sequence $S_{1:T}$ is hidden?

The key difficulty is that the model contains an unobserved sequence:
$$S_{1:T} = (S_1,\dots,S_T)$$
If that sequence were known, likelihood evaluation would be simple. But in real data, it is not known. Therefore, the observed-data likelihood must average over all possible hidden regime paths.
This section should therefore distinguish:
1. **complete-data likelihood**: likelihood if the hidden regimes were known;
2. **observed-data likelihood**: likelihood of the visible data after summing over all possible hidden regimes.
Do **not** fully derive the Hamilton filter here. That belongs in Section 3.5. In this section, the filter should appear only as the reason why the likelihood can be evaluated efficiently.
### Parameters used in the likelihood
The model parameters are:
$$\theta = (\pi, A, \mu_1,\dots,\mu_K,\sigma_1^2,\dots,\sigma_K^2)$$
Where:

| Symbol       | Meaning                     |
| ------------ | --------------------------- |
| $\pi$        | initial regime distribution |
| $A$          | transition matrix           |
| $\mu_j$      | mean of regime $j$          |
| $\sigma_j^2$ | variance of regime $j$      |
| $K$          | number of regimes           |


The likelihood asks:
$$p(y_{1:T}\mid \theta)$$
That is:
> If the parameters $\theta$ were fixed, how probable would the observed sequence $y_{1:T}$ be?

### Why the hidden states make the likelihood nontrivial
The observations are:
$$y_{1:T} = (y_1,\dots,y_T)$$
The hidden regimes are:
$$S_{1:T} = (S_1,\dots,S_T)$$
where:
$$S_t \in \{1,\dots,K\}$$
The difficulty is that $S_{1:T}$ is not observed.
For a single time point $t$, there are $K$ possible hidden regimes.
For the full sequence of length $T$, there are:
$$K^T$$
possible regime paths.
For example, if $K=2$ and $T=5$, possible paths include:
$$(1,1,1,1,1),$$$$(1,1,1,1,2),$$$$(1,1,1,2,1)$$
and so on.
There are:
$$2^5 = 32$$
possible paths.
If $K=2$ and $T=1000$, there are:
$$2^{1000}$$
possible hidden paths, which is computationally impossible to enumerate directly.
So the main idea of this section is:
> The observed likelihood is a sum over all possible hidden regime paths, but this sum cannot be computed naively.

#### Complete-data likelihood
First imagine an easier world.
Suppose the hidden state sequence were known:
$$S_{1:T} = (S_1,\dots,S_T)$$
Then the joint likelihood of observations and hidden regimes would be:
$$p(y_{1:T},S_{1:T}\mid\theta)$$
This is called the **complete-data likelihood**.
It is called “complete-data” because it treats both:
- the observed data $y_{1:T}$;
- the hidden regime data $S_{1:T}$;
as if they were available.
#### Factorization of the complete-data likelihood
The model has three components:
1. initial regime probability;
2. transition probabilities between regimes;
3. regime-conditional observation densities.
Therefore:
$$p(S_{1:T}\mid\theta) = p(S_1\mid\theta) \prod_{t=2}^T p(S_t\mid S_{t-1},\theta) \prod_{t=1}^T p(y_t\mid S_t,\theta)$$
Now substitute the model definitions.
Initial state:
$$p(S_1\mid\theta)=\pi_{S_1}$$
Transition:
$$p(S_t\mid S_{t-1},\theta) = A_{S_{t-1},S_t}$$
Observation density:
$$p(y_t\mid S_t,\theta) = f_{S_t}(y_t)$$
Therefore:
$$p(y_{1:T},S_{1:T}\mid\theta) = \pi_{S_1} \prod_{t=2}^T A_{S_{t-1},S_t} \prod_{t=1}^T f_{S_t}(y_t)$$
This is the central complete-data likelihood formula.
### Intuition behind the complete-data likelihood
The formula:
$$p(y_{1:T},S_{1:T}\mid\theta) = \pi_{S_1} \prod_{t=2}^T A_{S_{t-1},S_t} \prod_{t=1}^T f_{S_t}(y_t)$$
says that a hidden path is likely if three things are true.
#### It starts in a likely initial regime
The term:
$$\pi_{S_1}$$
measures how likely the first hidden regime is.
If:
$$S_1 = j$$
then the contribution is:
$$\pi_j$$
####  It follows likely transitions
The term:
$$St\prod_{t=2}^T A_{S_{t-1},S_t}$$
measures how likely the sequence of regime switches is.
If the path often uses high-probability transitions, this term is large.
If it uses many unlikely transitions, this term becomes small.
For example, if:
$$A_{12}=0.01$$
then a path that frequently switches from regime 1 to regime 2 is unlikely.
#### It explains the observations well
The term:
$$\prod_{t=1}^T f_{S_t}(y_t)$$
measures how well the selected regime at each time explains the corresponding observation.
If $S_t=j$, then the observation likelihood is:
$$f_j(y_t)$$
So if $y_t$ is very plausible under regime $j$, this contribution is relatively large.
If $y_t$ is very unlikely under regime $j$, this contribution is small.
### Log complete-data likelihood
Products of many probabilities are inconvenient, so we often work with logs.
Starting from:
$$p(y_{1:T},S_{1:T}\mid\theta) = \pi_{S_1} \prod_{t=2}^T A_{S_{t-1},S_t} \prod_{t=1}^T f_{S_t}(y_t)$$
take the logarithm:
$$\log p(y_{1:T},S_{1:T}\mid\theta) = \log \pi_{S_1} + \sum_{t=2}^T \log A_{S_{t-1},S_t} + \sum_{t=1}^T \log f_{S_t}(y_t)$$
This form is important because it separates the likelihood into additive parts:

|Term|Meaning|
|---|---|
|$\log \pi_{S_1}$|contribution of initial regime|
|$\sum_{t=2}^T \log A_{S_{t-1},S_t}$|contribution of regime transitions|
|$\sum_{t=1}^T \log f_{S_t}(y_t)$|contribution of observations|

This decomposition will later be useful for EM, because EM works with the expected complete-data log-likelihood
But in this section, the main point is simpler:
> If the hidden regime path were known, likelihood evaluation would be direct.

### But the complete-data likelihood is not directly available
In real data, we do **not** know:
$$S_{1:T}$$
So we cannot directly compute:
$$p(y_{1:T},S_{1:T}\mid\theta)$$
for the true path, because the true path is unknown.
Instead, the actual likelihood of the observed data is:
$$p(y_{1:T}\mid\theta)$$
This is the likelihood used for:
- model fitting;
- likelihood comparison;
- convergence monitoring;
- AIC/BIC;
- optimizer objective functions;
- EM log-likelihood history.
### Observed-data likelihood
Because $S_{1:T}$ is hidden, the observed-data likelihood marginalizes over all possible hidden paths:
$$p(y_{1:T}\mid\theta) = \sum_{S_{1:T}} p(y_{1:T},S_{1:T}\mid\theta)$$
More explicitly:
$$p(y_{1:T}\mid\theta) = \sum_{s_1=1}^K \sum_{s_2=1}^K \cdots \sum_{s_T=1}^K p(y_{1:T},S_1=s_1,\dots,S_T=s_T\mid\theta)$$
Using the complete-data factorization:
$$p(y_{1:T}\mid\theta) = \sum_{s_1=1}^K \cdots \sum_{s_T=1}^K \left[ \pi_{s_1} \prod_{t=2}^T A_{s_{t-1},s_t} \prod_{t=1}^T f_{s_t}(y_t) \right]$$
This is the mathematically correct likelihood of the observed sequence.
### Why naive summation is infeasible
The sum $\sum_{S_{1:T}}$ contains $K^T$ terms.
This grows exponentially with $T$.
For example:

|$K$|$T$|Number of paths|
|---|---|---|
|2|10|$2^{10}=1024$|
|2|100|$2^{100}$|
|3|100|$3^{100}$|
|4|1000|$4^{1000}$|

This is impossible to compute by explicit enumeration for realistic time-series lengths.
So the observed-data likelihood is mathematically a large sum, but computationally it must be evaluated recursively.
This is exactly why forward filtering is needed.

### Recursive likelihood evaluation idea
The forward filter avoids enumerating all paths by summarizing the past through a probability vector.
At each time $t$, define the one-step-ahead predictive density:
$$c_t = p(y_t \mid y_{1:t-1},\theta)$$
Then the observed-data likelihood can be factorized as:
$$p(y_{1:T}\mid\theta) = p(y_1\mid\theta) p(y_2\mid y_1,\theta) \cdots p(y_T\mid y_{1:T-1},\theta)$$
In compact notation:
$$p(y_{1:T}\mid\theta) = \prod_{t=1}^T p(y_t\mid y_{1:t-1},\theta)$$
Using:
$$c_t = p(y_t\mid y_{1:t-1},\theta)$$
we get:
$$p(y_{1:T}\mid\theta) = \prod_{t=1}^T c_t$$
Therefore:
$$\log p(y_{1:T}\mid\theta) = \sum_{t=1}^T \log c_t​$$
This is the log-likelihood computed by the filter.
### What is $c_t$?
The predictive density is:
$$c_t = p(y_t\mid y_{1:t-1},\theta)$$
It measures how likely the new observation is before updating with that observation.
Because the regime $S_t$ is hidden, we sum over possible regimes at time $t$:
$$c_t = \sum_{j=1}^K p(y_t,S_t=j\mid y_{1:t-1},\theta)$$
Using the product rule:
$$c_t = \sum_{j=1}^K p(y_t\mid S_t=j,y_{1:t-1},\theta) p(S_t=j\mid y_{1:t-1},\theta)$$
By the conditional observation assumption:
$$p(y_t\mid S_t=j,y_{1:t-1},\theta) = p(y_t\mid S_t=j,\theta) = f_j(y_t)$$
And define the predicted regime probability:
$$\alpha_{t|t-1}(j) = p(S_t=j\mid y_{1:t-1},\theta)$$
Therefore:
$$c_t = \sum_{j=1}^K f_j(y_t)\alpha_{t|t-1}(j)$$
This formula connects the observed likelihood to filtering.
But the detailed recursion for computing $\alpha_{t|t-1}(j)$ belongs in Section 3.5.
### Observed log-likelihood
The observed log-likelihood is:
$$\ell(\theta) = \log p(y_{1:T}\mid\theta)$$
Using the predictive factorization:
$$\ell(\theta) = \sum_{t=1}^T \log c_t.$$
This is the scalar objective that the model tries to increase during estimation.
In EM, one monitors:
$$\ell(\theta^{(m)})$$
across iterations.
Convergence can be checked using:
$$|\ell(\theta^{(m+1)})-\ell(\theta^{(m)})| < \varepsilon$$
### Why log-likelihood is used
The likelihood:
$$p(y_{1:T}\mid\theta) = \prod_{t=1}^T c_t$$
can become extremely small because it multiplies many densities.
For large $T$, direct multiplication is numerically unstable.
The log-likelihood solves this:
$$\log p(y_{1:T}\mid\theta) = \sum_{t=1}^T \log c_t​$$
This has three benefits:
1. sums are numerically more stable than products;
2. optimization is easier to monitor;
3. likelihood changes are easier to compare across iterations.
### Complete-data likelihood versus observed-data likelihood
This distinction is central.

|Object|Formula|Meaning|
|---|---|---|
|Complete-data likelihood|$p(y_{1:T},S_{1:T}\mid\theta)$|Likelihood if regimes were known|
|Observed-data likelihood|$p(y_{1:T}\mid\theta)$|Likelihood after summing over hidden regimes|

The complete-data likelihood is:
$$p(y_{1:T},S_{1:T}\mid\theta) = \pi_{S_1} \prod_{t=2}^T A_{S_{t-1},S_t} \prod_{t=1}^T f_{S_t}(y_t)$$
The observed-data likelihood is:
$$p(y_{1:T}\mid\theta) = \sum_{S_{1:T}} p(y_{1:T},S_{1:T}\mid\theta)$$
The complete-data likelihood is easier to write and interpret.
The observed-data likelihood is what we actually need.
### Why this distinction matters for EM
This section should briefly prepare for Section 3.8.
The hidden regime path is missing data.
If $S_{1:T}$ were observed, estimating parameters would be simpler:
- count initial states to estimate $\pi$;
- count transitions to estimate $A$;
- group observations by regime to estimate $\mu_j$ and $\sigma_j^2$.

But $S_{1:T}$ is hidden.
Therefore, EM works by using the expected complete-data log-likelihood:
$$Q(\theta\mid\theta^{(m)}) = \mathbb{E}_{S_{1:T}\mid y_{1:T},\theta^{(m)}} [ \log p(y_{1:T},S_{1:T}\mid\theta) ]$$

## 3.5 Forward filtering
### What this section should do
This section should explain the **Hamilton forward filter**.
The forward filter is the recursive algorithm that computes, at every time $t$, the posterior probability of each hidden regime using only the observations available up to that time.
The central quantity is:
$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$
This is called the **filtered probability**.
It answers:
> After observing $y_1,\dots,y_t$, what is the probability that the process is currently in regime $j$?

This is the main online inference object in a Markov switching model
#### What this section should not include
Keep the section focused.
Do **not** derive the backward smoother here. Only state that smoothing later uses filter outputs.
Do **not** derive pairwise transition probabilities here. Only mention that they require filtered and predicted probabilities.
Do **not** derive EM updates here. Only mention that the filter supplies likelihoods and E-step inputs.
Do **not** discuss Student-$t$ or other emission extensions here.
Do **not** deeply discuss model selection here.
The purpose is only:
```
Hamilton filtering recursion:prediction → predictive density → update → likelihood accumulation
```
### What filtering is not
Before defining the recursion, it is important to separate filtering from nearby concepts.
#### Filtering is not smoothing
Filtering uses only data up to time $t$:
$$\Pr(S_t=j \mid y_{1:t})$$
Smoothing uses the full sample:
$$\Pr(S_t=j \mid y_{1:T})$$
So filtering is online, while smoothing is retrospective.
Filtering answers:
> What would the model believe at time $t$ using information available then?

Smoothing answers:
> What does the model believe about time $t$ after seeing the future?

Smoothing belongs in Section 3.6. This section should focus only on filtering.
###### Filtering is not parameter estimation
Filtering assumes the parameters are already fixed:
$$\theta = (\pi,A,\mu_1,\dots,\mu_K,\sigma_1^2,\dots,\sigma_K^2)$$
Given $\theta$, the filter computes posterior regime probabilities and the likelihood.
Parameter estimation by EM uses filtering as one step, but EM itself belongs in Section 3.8.
So in this section, treat $\theta$ as known.
### Objects needed by the filter
The filter uses three components of the model.
#### Initial distribution
The initial regime distribution:
$$\pi_j = \Pr(S_1=j)$$
This initializes the recursion.
#### Transition matrix
The transition probabilities:
$$A_{ij} = \Pr(S_t=j \mid S_{t-1}=i)$$
These are used in the prediction step.
#### Emission density
The regime-dependent Gaussian density:
$$f_j(y_t) = p(y_t \mid S_t=j)$$
For your Gaussian model:
$$f_j(y_t) = \frac{1}{\sqrt{2\pi\sigma_j^2}} \exp \left( -\frac{(y_t-\mu_j)^2}{2\sigma_j^2} \right)$$
The filter evaluates this density for every regime $j$ and every observation $y_t$.
### The two probability vectors in filtering
The filter alternates between two related probability vectors.
#### Predicted probabilities
The predicted probability is:
$$\alpha_{t|t-1}(j) = \Pr(S_t=j \mid y_{1:t-1})$$
This is the model’s belief about the regime at time $t$ **before observing $y_t$**.
It uses:
- past observations $y_1,\dots,y_{t-1}$;
- the transition matrix $A$.
It does not yet use the current observation $y_t$.
#### Filtered probabilities
The filtered probability is:
$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$
This is the model’s belief about the regime at time $t$ **after observing $y_t$**.
It uses:
- past observations $y_1,\dots,y_{t-1}$;
- the current observation $y_t$;
- transition dynamics;
- emission likelihoods.
#### Intuition
At every time step, the filter performs:
```
previous belief → prediction → observe y_t → correction
```
Mathematically:
```
α_{t-1|t-1} → α_{t|t-1} → α_{t|t}
```
So the filter is a recursive Bayesian update.
### Initialization
At time $t=1$, before seeing $y_1$, the model starts from the initial distribution:
$$\alpha_{1|0}(j) = \Pr(S_1=j) = \pi_j$$
Then the first observation updates this prior:
$$c_1 = \sum_{j=1}^K f_j(y_1)\pi_j$$
The filtered probability at $t=1$ is:
$$\alpha_{1|1}(j) = \frac{ f_j(y_1)\pi_j }{ c_1 }$$
#### Intuition
At the first observation, there is no previous filtered belief. The model begins with $\pi$.
If $\pi_1$ is large, regime 1 starts with high prior probability.
But if $y_1$ is much more plausible under regime 2, then the update may shift probability toward regime 2.

### Prediction step
Suppose we already have the filtered probabilities from the previous time:
$$\alpha_{t-1|t-1}(i) = \Pr(S_{t-1}=i \mid y_{1:t-1})$$
Before observing $y_t$, we predict the regime distribution at time $t$.
The prediction equation is:
$$\alpha_{t|t-1}(j) = \sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i)$$
This is one of the central equations of the filter.
#### Derivation of the prediction step
Start from the definition:
$$\alpha_{t|t-1}(j) = \Pr(S_t=j \mid y_{1:t-1})$$
Use the law of total probability over the previous hidden state:
$$\Pr(S_t=j \mid y_{1:t-1}) = \sum_{i=1}^K \Pr(S_t=j,S_{t-1}=i \mid y_{1:t-1}$$
Apply the product rule:
$$= \sum_{i=1}^K \Pr(S_t=j \mid S_{t-1}=i,y_{1:t-1}) \Pr(S_{t-1}=i \mid y_{1:t-1})$$
By the Markov property:
$$\Pr(S_t=j \mid S_{t-1}=i,y_{1:t-1}) = \Pr(S_t=j \mid S_{t-1}=i) = A_{ij}$$
Also:
$$\Pr(S_{t-1}=i \mid y_{1:t-1}) = \alpha_{t-1|t-1}(i)$$
Therefore:
$$\alpha_{t|t-1}(j) = \sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i)$$
#### Intuition behind prediction
The model asks:
> What is the probability of being in regime $j$ today?

To answer, it considers every possible regime yesterday.
For each previous regime $i$:
1. how likely was yesterday’s regime $i$?
$$\alpha_{t-1|t-1}(i)$$
2. how likely is it to move from $i$ to $j$?
$$A_{ij}$$
Multiply these and sum over all possible previous regimes.
###### Matrix form
If $\alpha_{t-1|t-1}$ is treated as a row vector, then:
$$\alpha_{t|t-1} = \alpha_{t-1|t-1} A$$
This means the transition matrix propagates posterior belief forward by one time step.
#### Predictive density
After prediction, the model has:
$$alpha_{t|t-1}(j) = \Pr(S_t=j \mid y_{1:t-1})$$
Now it observes $y_t$.
Before updating the regime probabilities, the model computes how likely $y_t$ was overall.
This quantity is the predictive density:
$$c_t = p(y_t \mid y_{1:t-1})$$
Since the regime $S_t$ is hidden, we sum over possible regimes:
$$c_t = \sum_{j=1}^K p(y_t,S_t=j \mid y_{1:t-1})$$
Use the product rule:
$$c_t = \sum_{j=1}^K p(y_t \mid S_t=j,y_{1:t-1}) p(S_t=j \mid y_{1:t-1})$$
By the observation assumption:
$$p(y_t \mid S_t=j,y_{1:t-1}) = p(y_t \mid S_t=j) = f_j(y_t)$$
And:
$$p(S_t=j \mid y_{1:t-1}) = \alpha_{t|t-1}(j)$$
Therefore:
$$c_t = \sum_{j=1}^K f_j(y_t)\alpha_{t|t-1}(j)$$
### Intuition behind $c_t$
The predictive density is a weighted average of the regime-specific densities.
Each regime contributes:
$$f_j(y_t)$$
weighted by the probability that regime $j$ was expected before observing $y_t$:
$$\alpha_{t|t-1}(j)$$
So $c_t$ is high if:
- $y_t$ is likely under regimes that were predicted to be likely.
And $c_t$ is low if:
- $y_t$ is unlikely under all likely regimes.
This makes $-\log c_t$ a natural “surprise” score, although that application belongs more to an online monitoring section.
### Update step
After computing $c_t$, the filter updates the predicted probabilities using Bayes’ rule.
The filtered probability is:
$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$
Since:
$$y_{1:t} = (y_{1:t-1},y_t)$$
we write:
$$\Pr(S_t=j \mid y_{1:t}) = \Pr(S_t=j \mid y_t,y_{1:t-1})$$
By Bayes’ rule:
$$\Pr(S_t=j \mid y_t,y_{1:t-1}) = \frac{ p(y_t \mid S_t=j,y_{1:t-1}) \Pr(S_t=j \mid y_{1:t-1}) }{ p(y_t \mid y_{1:t-1}) }$$
Substitute the model quantities:
$$p(y_t \mid S_t=j,y_{1:t-1}) = f_j(y_t)$$
$$\Pr(S_t=j \mid y_{1:t-1}) = \alpha_{t|t-1}(j)$$
$$p(y_t \mid y_{1:t-1}) = c_t$$
Therefore:
$$\alpha_{t|t}(j) = \frac{ f_j(y_t)\alpha_{t|t-1}(j) }{ c_t }$$$$_t = \sum_{j=1}^K f_j(y_t)\alpha_{t|t-1}(j)$$
#### Update
$$\alpha_{t|t}(j) = \frac{ f_j(y_t)\alpha_{t|t-1}(j) }{ c_t }$$
This is the second central equation of the filter.
#### Intuition behind update
The posterior probability of regime $j$ is proportional to:
$$\text{predicted probability} \times \text{observation likelihood}$$
That is:
$$\alpha_{t|t}(j) \propto \alpha_{t|t-1}(j) f_j(y_t)$$
The denominator $c_t$ normalizes the probabilities so that:
$$\sum_{j=1}^K \alpha_{t|t}(j)=1$$
So a regime receives high posterior probability if:
1. it was plausible before observing $y_t$;
2. it explains $y_t$ well.
This is the core intuition of the Hamilton filter.
### Why the update is normalized
The numerator:
$$f_j(y_t)\alpha_{t|t-1}(j)$$
is an unnormalized regime weight.
Define:
$$w_j = f_j(y_t)\alpha_{t|t-1}(j)$$
Then:
$$c_t = \sum_{j=1}^K w_j$$
The update is:
$$alpha_{t|t}(j) = \frac{w_j}{\sum_{\ell=1}^K w_\ell}$$
Therefore:
$$\sum_{j=1}^K \alpha_{t|t}(j) = \sum_{j=1}^K \frac{w_j}{\sum_{\ell=1}^K w_\ell} = 1$$
This guarantees that the filtered vector remains a valid probability distribution.
### Full filtering recursion
For $t=1,\dots,T$, the filter runs:
#### Initialisation
$$\alpha_{1|0}(j)=\pi_j$$
#### Predictive density
$$c_1 = \sum_{j=1}^K f_j(y_1)\pi_j$$
#### First update
$$\alpha_{1|1}(j) = \frac{f_j(y_1)\pi_j}{c_1}$$
Then for $t=2,\dots,T$:
#### Prediction
$$\alpha_{t|t-1}(j) = \sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i)$$
#### Predictive density
$$cc_t = \sum_{j=1}^K f_j(y_t)\alpha_{t|t-1}(j)$$
#### Update
$$\alpha_{t|t}(j) = \frac{ f_j(y_t)\alpha_{t|t-1}(j) }{ c_t }$$
This is the complete forward filtering algorithm.
### Likelihood accumulation
The filter also computes the observed-data likelihood.
From Section 3.4:
$$p(y_{1:T}\mid\theta) = \prod_{t=1}^T c_t$$
Therefore, the log-likelihood is:
$$\ell(\theta) = \log p(y_{1:T}\mid\theta) = \sum_{t=1}^T \log c_t$$
The filter gets this “for free” because each $c_t$ is already computed for normalization.
### Why likelihood comes from the same recursion
The denominator of the Bayesian update is:
$$c_t = p(y_t\mid y_{1:t-1})$$
This is the probability density of observing $y_t$ given the past.
By the chain rule of probability:
$$p(y_{1:T}) = p(y_1) p(y_2\mid y_1) p(y_3\mid y_{1:2}) \cdots p(y_T\mid y_{1:T-1})$$
Each term is exactly $c_t$.
So:
$$p(y_{1:T}) = \prod_{t=1}^T c_t$$
Hence:
$$\log p(y_{1:T}) = \sum_{t=1}^T \log c_t$$
This is why the forward filter is both:
1. an inference algorithm for regimes;
2. a likelihood evaluation algorithm.
### Numerical stability and log-space computation
In exact mathematics, we can compute:
$$w_j = f_j(y_t)\alpha_{t|t-1}(j)$$
But in floating-point arithmetic, $f_j(y_t)$ can be very small.
If $y_t$ is far from $\mu_j$, then:
$$f_j(y_t) \approx 0$$
Multiplying many small densities can cause underflow.
Therefore, your implementation computes the unnormalized weights in log-space.
Define:
$$\log w_j = \log f_j(y_t) + \log \alpha_{t|t-1}(j)$$
Then:
$$\log c_t = \log \sum_{j=1}^K w_j = \log \sum_{j=1}^K \exp(\log w_j)$$
To compute this stably, use log-sum-exp:
$$\log \sum_{j=1}^K e^{x_j} = m + \log \sum_{j=1}^K e^{x_j-m}$$
where:
$$m=\max_j x_j$$
In this case:
$$x_j=\log w_j$$
### Recovering filtered probabilities from log-weights
Once:
$$\log c_t$$
has been computed, the filtered probability can be computed as:
$$\alpha_{t|t}(j) = \frac{w_j}{c_t}$$
In log form:
$$\log \alpha_{t|t}(j) = \log w_j - \log c_t$$
Therefore:
$$\alpha_{t|t}(j) = \exp(\log w_j - \log c_t)$$
This is numerically stable because:
$$\log w_j - \log c_t \le 0$$
for all $j$, so the exponentials are not exploding.
The resulting probabilities should sum to one, up to floating-point error.
### Filter as online inference
The filter is naturally online because it only needs:
1. previous filtered probability vector;
2. transition matrix;
3. current observation;
4. emission densities.
At time $t$, it does not need future observations.
That makes:
$$\alpha_{t|t}(j)$$
appropriate for online regime monitoring.
### Computational complexity
For each time step $t$, the prediction step computes:
$$\alpha_{t|t-1}(j) = \sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i)$$
for each $j$.
This costs:
$$O(K^2)$$
per time step.
The emission/update step evaluates $K$ densities and normalizes $K$ weights, which costs:
$$O(K).$$
So the full filter costs:
$$O(TK^2)$$
For small $K$, this is efficient.

### Key conceptual summary
The forward filter has three equations.
Prediction:
$$\alpha_{t|t-1}(j) = \sum_{i=1}^K A_{ij}\alpha_{t-1|t-1}(i)$$
Predictive density:
$$c_t = \sum_{j=1}^K f_j(y_t)\alpha_{t|t-1}(j)$$
Update:
$$\alpha_{t|t}(j) = \frac{ f_j(y_t)\alpha_{t|t-1}(j) }{ c_t }$$
The likelihood follows as:
$$\ell(\theta) = \sum_{t=1}^T \log c_t$$
In one sentence:
> Forward filtering recursively combines Markovian regime prediction with observation-based Bayesian correction, producing both online posterior regime probabilities and the observed-data log-likelihood.

## 3.6 Backward smoothing
### What this section should do
Section 3.5 explained **forward filtering**.
Filtering computes:
$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$
This is the posterior probability of regime $j$ at time $t$ using only observations up to time $t$.
Section 3.6 should explain **backward smoothing**.
Smoothing computes:
$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$
This is the posterior probability of regime $j$ at time $t$ using the **entire observed sequence**.
The purpose of this section is:
> To show how filtered regime probabilities can be refined by using future observations.

### What smoothing assumes is already known
Backward smoothing is not the first inference pass.
It uses quantities already produced by the forward filter:

| Quantity     | Meaning                | Produced by                    |
| ------------ | ---------------------- | ------------------------------ |
| $\alpha_{t   | t}(i)$                 | filtered probability           |
| $\alpha_{t+1 | t}(j)$                 | one-step predicted probability |
| $A_{ij}$     | transition probability | model parameters               |

The smoother also needs the next smoothed probability:
$$\gamma_{t+1}(j)$$
So the computation runs backward:
$$T,\ T-1,\ T-2,\ \dots,\ 1$$
That is important:
> Smoothing does not recompute Gaussian densities. It reuses the filtered and predicted probabilities already computed by the forward pass.

### Main smoothing quantity
Define the smoothed posterior:
$$\gamma_t(i) = \Pr(S_t=i \mid y_{1:T})$$
This is the probability that the latent state at time $t$ was regime $i$, after observing the whole sample.
For each fixed $t$, the vector:
$$\gamma_t = (\gamma_t(1),\dots,\gamma_t(K))$$
is a probability distribution, so:
$$\gamma_t(i) \ge 0$$
and:
$$\sum_{i=1}^K \gamma_t(i)=1$$
#### Intuition
The smoother gives a retrospective classification distribution.
It does not say:
$$S_t=i$$
with certainty.
It says:
$$\Pr(S_t=i \mid y_{1:T})$$
is high or low.
So, in thesis language, prefer:
> The smoothed posterior assigns high probability to regime $i$ at time $t$

rather than:
> The model proves that time $t$ was regime $i$.
### Terminal condition
At the final time point $T$, there are no future observations.
Therefore, smoothing and filtering coincide:
$$\gamma_T(j) = \Pr(S_T=j \mid y_{1:T})$$
But the filtered probability at time $T$ is already:
$$\alpha_{T|T}(j) = \Pr(S_T=j \mid y_{1:T})$$
Therefore:
$$\gamma_T(j) = \alpha_{T|T}(j)$$
This is the terminal condition of the backward recursion.
#### Intuition
At the last observation, there is no future left to revise the filtering estimate.
So the final smoothed probability equals the final filtered probability.
### Backward smoothing recursion
For:
$$t=T-1,T-2,\dots,1$$
the smoother computes:
$$\gamma_t(i) = \alpha_{t|t}(i) \sum_{j=1}^K \frac{ A_{ij}\gamma_{t+1}(j) }{ \alpha_{t+1|t}(j) }$$
This is the main equation of this section.
It updates the probability of state $i$ at time $t$ using:
1. the filtered probability at time $t$;
2. the transition probabilities from $i$ to future states $j$;
3. the smoothed probabilities at time $t+1$;
4. the predicted probabilities at time $t+1$.
This is often called the Kim smoother or Hamilton-Kim backward smoother in Markov-switching models. Cite it as:
```
\cite{Kim1994,Hamilton1994}
```
### Deriving the recursion
The goal is:
$$\gamma_t(i) = \Pr(S_t=i \mid y_{1:T})$$
Split the full sample into:
$$y_{1:T} = (y_{1:t},y_{t+1:T})$$
The filtered probability already gives:
$$\alpha_{t|t}(i) = \Pr(S_t=i \mid y_{1:t})$$
The smoother asks how future observations modify that belief.
A useful identity is:
$$\gamma_t(i) = \sum_{j=1}^K \Pr(S_t=i,S_{t+1}=j \mid y_{1:T})$$
That is, to get the posterior probability of $S_t=i$, sum over all possible next states $S_{t+1}=j$.
Now decompose the joint posterior:
$$\Pr(S_t=i,S_{t+1}=j \mid y_{1:T}) = \Pr(S_t=i \mid S_{t+1}=j,y_{1:T}) \Pr(S_{t+1}=j \mid y_{1:T})$$
The second term is:
$$\Pr(S_{t+1}=j \mid y_{1:T}) = \gamma_{t+1}(j)$$
So:
$$\gamma_t(i) = \sum_{j=1}^K \Pr(S_t=i \mid S_{t+1}=j,y_{1:T}) \gamma_{t+1}(j)$$
The key backward probability can be expressed using filtered and predicted quantities:
$$\Pr(S_t=i \mid S_{t+1}=j,y_{1:t}) = \frac{ \Pr(S_{t+1}=j \mid S_t=i,y_{1:t}) \Pr(S_t=i \mid y_{1:t}) }{ \Pr(S_{t+1}=j \mid y_{1:t}) }$$
Now substitute the model quantities.
Because of the Markov property:
$$\Pr(S_{t+1}=j \mid S_t=i,y_{1:t}) = A_{ij}$$
The filtered probability is:
$$\Pr(S_t=i \mid y_{1:t}) = \alpha_{t|t}(i)$$
The predicted probability is:
$$\Pr(S_{t+1}=j \mid y_{1:t}) = \alpha_{t+1|t}(j)$$
Therefore:
$$\Pr(S_t=i \mid S_{t+1}=j,y_{1:t}) = \frac{ A_{ij}\alpha_{t|t}(i) }{ \alpha_{t+1|t}(j) }$$
Plug this into the sum:
$$\gamma_t(i) = \sum_{j=1}^K \frac{ A_{ij}\alpha_{t|t}(i) }{ \alpha_{t+1|t}(j) } \gamma_{t+1}(j)$$
Since $\alpha_{t|t}(i)$ does not depend on $j$, factor it out:
$$\gamma_t(i) = \alpha_{t|t}(i) \sum_{j=1}^K \frac{ A_{ij}\gamma_{t+1}(j) }{ \alpha_{t+1|t}(j) }$$
This is the smoother recursion.
### Intuition behind each term
The recursion is:
$$\gamma_t(i) = \alpha_{t|t}(i) \sum_{j=1}^K \frac{ A_{ij}\gamma_{t+1}(j) }{ \alpha_{t+1|t}(j) }$$
Each part has a clear meaning.
#### Filtered belief at time $t$
$$\alpha_{t|t}(i)$$
This is what the model believed about state $i$ at time $t$ before seeing future data.
The smoother starts from this belief.
If:
$$\alpha_{t|t}(i)$$
is already near zero, then the smoothed probability will usually also be small.
#### Transition probability into future state
$$A_{ij}$$
This says how plausible it is to move from state $i$ at time $t$ to state $j$ at time $t+1$.
If future data strongly suggest state $j$ at $t+1$, then state $i$ at $t$ becomes more plausible if
$$A_{ij}$$
is large.
#### Future smoothed probability
$$\gamma_{t+1}(j)$$
This is the posterior probability of state $j$ at time $t+1$ using the whole sample.
It carries information from the future backward.
If:
$$\gamma_{t+1}(j)$$
is large, then the model believes state $j$ was likely at $t+1$.
That affects which states are plausible at $t$.
#### Predicted probability normalization
$$\alpha_{t+1|t}(j)$$
This is what the forward filter predicted for state $j$ at time $t+1$ before seeing $y_{t+1}$.
The ratio:
$$\frac{\gamma_{t+1}(j)}{\alpha_{t+1|t}(j)}$$
measures how much the full future evidence changed belief about state $j$ relative to the one-step prediction.
If:
$$\gamma_{t+1}(j) > \alpha_{t+1|t}(j)$$
then future observations made state $j$ more likely than expected.
If:
$$\gamma_{t+1}(j) < \alpha_{t+1|t}(j)$$
then future observations made state $j$ less likely than expected.
So smoothing propagates this correction backward through the transition matrix.
### A compact alternative view
The recursion can also be written as:
$$\gamma_t(i) = \alpha_{t|t}(i) \sum_{j=1}^K A_{ij} r_{t+1}(j)$$
where:
$$r_{t+1}(j) = \frac{\gamma_{t+1}(j)}{\alpha_{t+1|t}(j)}$$
Here:
$$r_{t+1}(j)$$
is a future correction factor.
Then:
$$\sum_{j=1}^K A_{ij}r_{t+1}(j)$$
says:
> If we were in state $i$ at time $t$, how compatible is that with the future corrections at time $t+1$?

This form is often easier to understand.
### Why smoothing is not online
Smoothing requires:
$$y_{1:T}$$
At time $t$, if future observations:
$$y_{t+1},\dots,y_T$$
are not yet available, then:
$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$
cannot be computed.
Therefore, smoothing is an offline procedure.
In live use, one uses filtered probabilities:
$$\alpha_{t|t}(j)$$
In historical analysis and training, one uses smoothed probabilities:
$$\gamma_t(j)$$
This distinction is important for writing about experiments.
### Numerical safeguards
The smoother recursion contains the denominator:
$$\alpha_{t+1|t}(j)$$
If this predicted probability is extremely small, then the ratio:
$$\frac{\gamma_{t+1}(j)}{\alpha_{t+1|t}(j)}$$
can become numerically unstable.
Mathematically, if the model is well-defined and probabilities are positive, this denominator should not be problematic. But in floating-point arithmetic, probabilities can become extremely small.
### Renormalization
After computing:
$$\tilde {\gamma}_t(i) = \alpha_{t|t}(i) \sum_{j=1}^K \frac{ A_{ij}\gamma_{t+1}(j) }{ \alpha_{t+1|t}(j) },$$
the implementation may renormalize:
$$\gamma_t(i) = \frac{ \tilde{\gamma}_t(i) }{ \sum_{\ell=1}^K \tilde{\gamma}_t(\ell) }$$
This ensures:
$$\sum_{i=1}^K \gamma_t(i)=1$$
In exact arithmetic, the recursion should preserve normalization. In floating-point arithmetic, small errors accumulate, so renormalization is useful.
#### Thesis phrasing
You can write:
> In the implementation, each smoothed probability vector is renormalized to remain on the probability simplex.

This is enough. No need to overdevelop numerical analysis in this section.

## 3.7 Pairwise transition probabilities
### What this section should do
Sections 3.5 and 3.6 gave two kinds of posterior probabilities:
Filtered probabilities:
$$\alpha_{t|t}(j) = \Pr(S_t=j \mid y_{1:t})$$
and smoothed probabilities:
$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$
These tell us how likely each **individual regime** is at each time point.
But for estimating the transition matrix $A$, this is not enough.
The transition matrix contains probabilities of moving from one regime to another:
$$A_{ij} = \Pr(S_t=j \mid S_{t-1}=i)$$
So to estimate $A_{ij}$, we need information about **pairs of adjacent states**:
$$(S_{t-1}, S_t)$$
That is the purpose of this section.
The central object is:
$$\xi_t(i,j) = \Pr(S_{t-1}=i, S_t=j \mid y_{1:T})$$
This is the **pairwise posterior transition probability**.
It answers:
> Given the entire observed sequence, what is the probability that the hidden process moved from regime $i$ at time $t-1$ to regime $j$ at time $t$?
### What this section should not include
Keep this section focused.
Do **not** rederive the full forward filter. Only refer to $\alpha_{t-1|t-1}$ and $\alpha_{t|t-1}$.
Do **not** rederive smoothing. Only use $\gamma_t$.
Do **not** derive the full EM algorithm. Only explain why $\xi_t$ is needed for the transition update.
Do **not** discuss Gaussian mean/variance updates. Those belong in Section 3.8.
Do **not** discuss model selection, diagnostics, or Student-$t$ emissions.
The purpose is only:
```
smoothed state probabilities are marginals;transition estimation needs adjacent-state joint probabilities;xi_t(i,j) provides those probabilities;expected transition counts are sums of xi_t(i,j).
```
### Why smoothed probabilities alone are not enough

The smoothed posterior:
\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})
tells us how likely regime $j$ was at time $t$.
For example:
\gamma_t = (0.1, 0.9)
means that tme $t$ was probably regime 2.
But the transition matrix asks a different question.
It asks:
> If the process was in regime $i$ at time $t-1$, how likely was it to move to regime $j$ at time $t$?

That is a joint question about two adjacent times.
To estimate:
A_{ij}​,
we need to know how often the model believes transitions of type:
i \to j
occurred.
The smoothed marginal probabilities $\gamma_t(j)$ do not directly tell us that. They only describe one time point at a time.
So we introduce:
\xi_t(i,j) = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T})
### Definition of pairwise transition probability
For:
t=2,\dots,T
define:
\xi_t(i,j) = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T})
Here:
- $i$ is the regime at time $t-1$;
- $j$ is the regime at time $t$;
- $y_{1:T}$ is the full observed sequence.
So $\xi_t(i,j)$ is a posterior probability over adjacent hidden-state pairs.
For each fixed time $t$, the values:
$$\{\xi_t(i,j)\}_{i=1,j=1}^{K,K}$$
form a probability distribution over all possible transitions between $t-1$ and $t$.
Therefore:
$$\xi_t(i,j) \ge 0$$
and:
$$\sum_{i=1}^K \sum_{j=1}^K \xi_t(i,j)=1$$
#### Intuition
At every transition time $t$, the model considers all possible movements:
$$1 \to 1,\quad 1 \to 2,\quad \dots,\quad K \to K$$
Then it assigns posterior probability to each possible movement.
For $K=2$, there are four possible transitions:

|Transition|Meaning|
|---|---|
|$1 \to 1$|stayed in regime 1|
|$1 \to 2$|switched from 1 to 2|
|$2 \to 1$|switched from 2 to 1|
|$2 \to 2$|stayed in regime 2|

At each $t$, $\xi_t(i,j)$ tells us how much posterior probability the model assigns to each of these four possibilities.
### Formula used in the model
The formula is:
$$\xi_t(i,j) = \frac{ \alpha_{t-1|t-1}(i) A_{ij} \gamma_t(j) }{ \alpha_{t|t-1}(j) }, \qquad t=2,\dots,T$$
This is the pairwise posterior transition probability used in your model layer.
The formula combines:

|Term|Meaning|
|---|---|
|$\alpha_{t-1|t-1}(i)$|
|$A_{ij}$|transition probability from $i$ to $j$|
|$\gamma_t(j)$|smoothed probability of regime $j$ at time $t$|
|$\alpha_{t|t-1}(j)$|
###  Deriving the pairwise formula
We want:
$$\xi_t(i,j) = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T})$$
Start by using the product rule:
$$\Pr(S_{t-1}=i,S_t=j \mid y_{1:T}) = \Pr(S_{t-1}=i \mid S_t=j, y_{1:T}) \Pr(S_t=j \mid y_{1:T})$$
The second term is the smoothed posterior:
$$\Pr(S_t=j \mid y_{1:T}) = \gamma_t(j)$$
So:
$$\xi_t(i,j) = \Pr(S_{t-1}=i \mid S_t=j, y_{1:T}) \gamma_t(j)$$
Now consider the backward conditional probability:
$$\Pr(S_{t-1}=i \mid S_t=j, y_{1:T})$$
Because future observations after time $t$ are already summarized through $\gamma_t(j)$, the backward conditional can be expressed using the filtered probability at $t-1$, the transition probability, and the one-step predicted probability:
$$\Pr(S_{t-1}=i \mid S_t=j, y_{1:t-1}) = \frac{ \Pr(S_t=j \mid S_{t-1}=i, y_{1:t-1}) \Pr(S_{t-1}=i \mid y_{1:t-1}) }{ \Pr(S_t=j \mid y_{1:t-1}) }$$
Now substitute the model quantities.
By the Markov property:
$$\Pr(S_t=j \mid S_{t-1}=i, y_{1:t-1}) = \Pr(S_t=j \mid S_{t-1}=i) = A_{ij}$$
By definition of filtered probability:
$$\Pr(S_{t-1}=i \mid y_{1:t-1}) = \alpha_{t-1|t-1}(i)$$
By definition of predicted probability:
$$\Pr(S_t=j \mid y_{1:t-1}) = \alpha_{t|t-1}(j)$$
Therefore:
$$\Pr(S_{t-1}=i \mid S_t=j, y_{1:t-1}) = \frac{ A_{ij}\alpha_{t-1|t-1}(i) }{ \alpha_{t|t-1}(j) }$$
Multiplying by $\gamma_t(j)$ gives:
$$\xi_t(i,j) = \frac{ \alpha_{t-1|t-1}(i) A_{ij} \gamma_t(j) }{ \alpha_{t|t-1}(j) }$$
This is the required formula.
### Intuition behind the formula
The formula is:
$$\xi_t(i,j) = \frac{ \alpha_{t-1|t-1}(i) A_{ij} \gamma_t(j) }{ \alpha_{t|t-1}(j) }$$
Read it as:
> Posterior probability of transition $i \to j$ equals the probability of being in $i$ at the previous time, times the probability of moving from $i$ to $j$, corrected by the full-sample posterior probability of being in $j$ at the current time.

Now examine each term.
#### Previous filtered probability
$$\alpha_{t-1|t-1}(i)$$
This says how likely regime $i$ was at time $t-1$ after observing data up to $t-1$.
If this is small, then the transition from $i$ to anything should be unlikely.
Example:
$$\alpha_{t-1|t-1}(i)=0.01$$
means the model barely believed it was in regime $i$ at time $t-1$, so transitions out of $i$ should receive little posterior mass.
#### Transition probability
$$A_{ij}$$
This is the model’s prior probability of moving from $i$ to $j$.
If:
$$A_{ij}$$
is large, then the transition $i \to j$ is structurally plausible.
If:
$$A_{ij}​$$
is small, then the transition is structurally unlikely.
So the pairwise posterior respects the Markov dynamics.
#### Current smoothed probability
$$\gamma_t(j)$$
This says how likely regime $j$ is at time $t$ after observing the entire sequence.
If:
$$\gamma_t(j)$$
is large, then transitions into $j$ should collectively receive large posterior probability.
If:
$$\gamma_t(j)$$
is small, then transitions into $j$ should collectively receive little posterior probability.
#### Predicted probability denominator
$$\alpha_{t|t-1}(j)$$
This is the total predicted probability of being in regime $j$ at time $t$, before seeing $y_t$.
Recall:
$$\alpha_{t|t-1}(j) = \sum_{\ell=1}^K A_{\ell j}\alpha_{t-1|t-1}(\ell)$$
So the denominator normalizes the contribution from source state $i$ relative to all possible source states that could lead into $j$.
In other words:
$$\frac{ \alpha_{t-1|t-1}(i)A_{ij} }{ \alpha_{t|t-1}(j) }$$
is the posterior probability that the previous state was $i$, given that the current state is $j$ and the past observations are known.
Then multiplying by $\gamma_t(j)$ allocates the smoothed probability of current state $j$ across possible previous states $i$.
### A useful interpretation: splitting $\gamma_t(j)$ by source state
For a fixed current state $j$, the smoothed probability:
$$\gamma_t(j)$$
tells us the total posterior probability that time $t$ is in regime $j$.
But this probability can be decomposed by previous state:
$$\gamma_t(j) = \sum_{i=1}^K \xi_t(i,j)$$
So the pairwise probabilities answer:
> Of the total posterior probability that time $t$ is regime $j$, how much came from previous regime $i$?
### Structural consistency properties
The pairwise probabilities must satisfy several consistency conditions.
These are useful both mathematically and as implementation checks.
#### Non-negativity
For all $i,j,t$:
$$\xi_t(i,j)\ge 0$$
This must hold because $\xi_t(i,j)$ is a probability.
#### Total probability equals one
For each fixed $t$:
$$\sum_{i=1}^K\sum_{j=1}^K \xi_t(i,j)=1$$
Why?
Because at time interval $(t-1,t)$, exactly one pair of states occurred:
$$(S_{t-1},S_t)=(i,j)$$
for some $i,j$.
The pairwise probabilities form a full probability distribution over all possible adjacent state pairs.
#### Column marginal
For each current state $j$:
$$\sum_{i=1}^K \xi_t(i,j) = \gamma_t(j)$$
Why?
Because summing over all possible previous states gives the probability of the current state:
$$\sum_i \Pr(S_{t-1}=i,S_t=j \mid y_{1:T}) = \Pr(S_t=j \mid y_{1:T})$$
So:
$$\sum_i \xi_t(i,j)=\gamma_t(j)$$
This confirms that the pairwise probabilities agree with the smoothed posterior at time $t$.
#### Row marginal
For each previous state $i$:
$$\sum_{j=1}^K \xi_t(i,j) = \gamma_{t-1}(i)$$
Why?
Because summing over all possible current states gives the probability of the previous state:
$$\sum_j \Pr(S_{t-1}=i,S_t=j \mid y_{1:T}) = \Pr(S_{t-1}=i \mid y_{1:T})$$
So:
$$\sum_j \xi_t(i,j)=\gamma_{t-1}(i)$$
This confirms that the pairwise probabilities agree with the smoothed posterior at time $t-1$.
### Expected transition counts
The pairwise posterior probabilities are used to estimate how many transitions of each type occurred.
Define the expected number of transitions from regime $i$ to regime $j$:
$$N_{ij}^{\mathrm{exp}} = \sum_{t=2}^T \xi_t(i,j)$$
This is not an observed count.
It is an expected count under the posterior distribution of the hidden states.
#### Intuition
If the hidden states were observed, the number of transitions from $i$ to $j$ would be:
$$N_{ij} = \sum_{t=2}^T \mathbf{1}\{S_{t-1}=i,S_t=j\}$$
But the states are hidden.
So EM replaces the indicator with its posterior expectation:
$$\mathbb{E} [ \mathbf{1}\{S_{t-1}=i,S_t=j\} \mid y_{1:T} ] = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T}) = \xi_t(i,j)$$
Therefore:
$$N_{ij}^{\mathrm{exp}} = \sum_{t=2}^T \mathbb{E} [ \mathbf{1}\{S_{t-1}=i,S_t=j\} \mid y_{1:T} ]$$
So expected transition counts are soft versions of ordinary transition counts.
### Connection to transition-matrix estimation
The M-step update for the transition matrix is:
$$A_{ij}^{(m+1)} = \frac{ N_{ij}^{\mathrm{exp}} }{ \sum_{j'=1}^K N_{ij'}^{\mathrm{exp}} }$$
Equivalently:
$$A_{ij}^{(m+1)} = \frac{ \sum_{t=2}^T \xi_t(i,j) }{ \sum_{t=2}^T \sum_{j'=1}^K \xi_t(i,j') }$$
Using the row marginal property:
$$\sum_{j'=1}^K \xi_t(i,j') = \gamma_{t-1}(i)$$
we can write:
$$A_{ij}^{(m+1)} = \frac{ \sum_{t=2}^T \xi_t(i,j) }{ \sum_{t=2}^T \gamma_{t-1}(i) }$$
Changing the index in the denominator:
$$\sum_{t=2}^T \gamma_{t-1}(i) = \sum_{t=1}^{T-1} \gamma_t(i)$$
So:
$$A_{ij}^{(m+1)} = \frac{ \sum_{t=2}^T \xi_t(i,j) }{ \sum_{t=1}^{T-1} \gamma_t(i) }$$
This is the transition update used in EM.
#### Intuition
The numerator is:
> expected number of transitions from $i$ to $j$.

The denominator is:
> expected number of times the process was in regime $i$ and had a chance to transition.

So the transition probability estimate is:
$$\frac{ \text{expected count of } i\to j }{ \text{expected count of all transitions out of } i }.$$
This is exactly how one would estimate a Markov transition matrix if the hidden states were known, except that observed counts are replaced by posterior expected counts.
### Why pairwise probabilities are the bridge between smoothing and EM
The smoother gives state occupancy probabilities:
$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$
These are enough to estimate Gaussian emission parameters:
$$\mu_j,\sigma_j^2$$
But they are not enough to estimate the transition matrix.
To estimate $A$, we need posterior information about adjacent states:
$$(S_{t-1},S_t)$$
That information is exactly:
$$\xi_t(i,j)$$
So the inference chain is:
```
filter → smoother → pairwise → transition update
```
###  Pairwise probabilities as soft transition labels
It helps to compare the hidden-state known and hidden-state unknown cases.
#### If hidden states were known
At each transition time $t$, there would be one true transition:
$$S_{t-1}=i,\quad S_t=j$$
The transition indicator would be:
$$\mathbf{1}\{S_{t-1}=i,S_t=j\} = \begin{cases} 1, & \text{if transition } i\to j \text{ occurred},\\ 0, & \text{otherwise}. \end{cases}$$
#### When hidden states are unknown
The model cannot observe the indicator.
So it computes the posterior probability:
$$\xi_t(i,j) = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T})$$
This is a soft transition label.
Instead of saying:
> at time $t$, the transition definitely was $1\to2$,

the model says:
> at time $t$, the transition $1\to2$ has posterior probability $0.23$.

Then the expected transition count accumulates these soft labels across time.
### Relationship to complete-data log-likelihood
This section is also connected to the complete-data likelihood from Section 3.4.
The transition part of the complete-data log-likelihood is:
$$\sum_{t=2}^T \log A_{S_{t-1},S_t}$$
Using indicators, this can be written as:
$$\sum_{t=2}^T \sum_{i=1}^K \sum_{j=1}^K \mathbf{1}\{S_{t-1}=i,S_t=j\} \log A_{ij}$$
In EM, the hidden transition indicator is replaced by its posterior expectation:
$$\mathbb{E} [ \mathbf{1}\{S_{t-1}=i,S_t=j\} \mid y_{1:T} ] = \xi_t(i,j)$$
So the expected transition part becomes:
$$\sum_{t=2}^T \sum_{i=1}^K \sum_{j=1}^K \xi_t(i,j) \log A_{ij}$$
This is why $\xi_t(i,j)$ is exactly the sufficient posterior quantity needed for updating $A$.

### Key conceptual summary
Smoothed probabilities give state marginals:
$$\gamma_t(j) = \Pr(S_t=j \mid y_{1:T})$$
Pairwise probabilities give transition marginals:
$$\xi_t(i,j) = \Pr(S_{t-1}=i,S_t=j \mid y_{1:T})$$

$$\xi_t(i,j) = \frac{ \alpha_{t-1|t-1}(i) A_{ij} \gamma_t(j) }{ \alpha_{t|t-1}(j) }$$

$$N_{ij}^{\mathrm{exp}} = \sum_{t=2}^T \xi_t(i,j)$$

> Pairwise transition probabilities convert posterior regime uncertainty into expected transition counts, which are the quantities needed to estimate the Markov transition matrix.

## 3.8 Parameter estimation by EM
### What this section should do
This section explains how the model parameters are estimated when the regime sequence is hidden.
The observed data are:

$$
y_{1:T} = (y_1,\dots,y_T)
$$
but the hidden regime path is not observed:
$$
S_{1:T} = (S_1,\dots,S_T)
$$
The parameter vector is:
$$
\theta = (\pi,A,\mu_1,\dots,\mu_K,\sigma_1^2,\dots,\sigma_K^2)
$$
where:

| Parameter | Meaning |
|---|---|
| $\pi$ | initial regime distribution |
| $A$ | transition matrix |
| $\mu_j$ | mean of regime $j$ |
| $\sigma_j^2$ | variance of regime $j$ |

If the hidden states were known, estimating the model would be straightforward:
- estimate $\pi$ from the first hidden state;
- estimate $A$ by counting transitions;
- estimate $\mu_j$ and $\sigma_j^2$ from observations assigned to regime $j$.
But the hidden states are not known.
Therefore, the model uses the **Expectation-Maximization algorithm**.
The basic idea is:
```
infer hidden states probabilistically → update parameters using those probabilities
```
At iteration $m$, EM alternates between:
1. **E-step**: compute posterior probabilities of hidden states and transitions under current parameters $\theta^{(m)}$;
2. **M-step**: update $\theta$ using those posterior probabilities as soft assignments.
The central quantities are:
$$
\gamma_t^{(m)}(j)
=
\Pr(S_t=j \mid y_{1:T};\theta^{(m)})
$$
and:
$$
\xi_t^{(m)}(i,j)
=
\Pr(S_{t-1}=i,S_t=j \mid y_{1:T};\theta^{(m)})
$$
The first gives posterior regime occupancy.
The second gives posterior transition probabilities.
Together, they allow the algorithm to update:
$$
\pi,\quad A,\quad \mu_j,\quad \sigma_j^2
$$
without observing the true hidden path.

Keep this section focused on parameter estimation.
Do **not** rederive the full forward filter. Only state that it computes likelihoods and provides inputs for smoothing.
Do **not** rederive backward smoothing. Only state that it computes $\gamma_t(j)$.
Do **not** rederive pairwise probabilities. Only state that they compute $\xi_t(i,j)$ and expected transition counts.
Do **not** discuss regime interpretation in detail. That belongs in Section 3.9.
Do **not** discuss AIC, BIC, model selection, diagnostics, or Student-$t$ extensions.
The purpose is only:
```
hidden regimes are missing data;  
EM replaces unknown state and transition indicators with posterior probabilities;  
the M-step updates π, A, μ, and σ² using soft counts and weighted moments.
```
### Why EM is needed
The observed-data likelihood is:
$$
p(y_{1:T}\mid\theta)
$$
Because the hidden state sequence is unobserved, this likelihood marginalizes over all possible hidden paths:
$$
p(y_{1:T}\mid\theta)
=
\sum_{S_{1:T}}
p(y_{1:T},S_{1:T}\mid\theta)
$$
The sum contains:
$$
K^T
$$
possible hidden regime paths.
The forward filter can evaluate this likelihood efficiently, but directly maximizing it is still difficult because the hidden states are not observed.
EM handles this by treating:
$$
S_{1:T}
$$
as missing data.
Instead of maximizing the observed-data likelihood directly, EM maximizes the expected complete-data log-likelihood.
### Complete-data likelihood reminder
If the hidden path were known, the complete-data likelihood would be:
$$
p(y_{1:T},S_{1:T}\mid\theta)
=
\pi_{S_1}
\prod_{t=2}^T A_{S_{t-1},S_t}
\prod_{t=1}^T f_{S_t}(y_t)
$$
Taking logarithms:
$$
\log p(y_{1:T},S_{1:T}\mid\theta)
=
\log \pi_{S_1}
+
\sum_{t=2}^T \log A_{S_{t-1},S_t}
+
\sum_{t=1}^T \log f_{S_t}(y_t)
$$
This separates into three parts:

| Part | Parameters involved |
|---|---|
| initial-state term | $\pi$ |
| transition term | $A$ |
| observation-density term | $\mu_j,\sigma_j^2$ |

This separation is what makes closed-form EM updates possible.
### Indicator-variable representation
To write the complete-data log-likelihood precisely, introduce state indicators:
$$
z_t(j)
=
\mathbf{1}\{S_t=j\}
$$
So:
$$
z_t(j)
=
\begin{cases}
1, & S_t=j,\\
0, & S_t\ne j.
\end{cases}
$$
Also introduce transition indicators:
$$
z_t(i,j)
=
\mathbf{1}\{S_{t-1}=i,S_t=j\},
\qquad t=2,\dots,T
$$
Then the complete-data log-likelihood becomes:
$$
\log p(y_{1:T},S_{1:T}\mid\theta)
=
\sum_{j=1}^K z_1(j)\log \pi_j
+
\sum_{t=2}^T
\sum_{i=1}^K
\sum_{j=1}^K
z_t(i,j)\log A_{ij}
+
\sum_{t=1}^T
\sum_{j=1}^K
z_t(j)\log f_j(y_t)
$$
#### Intuition
If the hidden states were observed, the indicators would be hard zeros and ones.
For example, if:
$$
S_t=2
$$
then:
$$
z_t(2)=1
$$
and all other $z_t(j)$ are zero.
But because the hidden state is unknown, EM replaces these hard indicators with posterior probabilities.
### Expected complete-data log-likelihood
At EM iteration $m$, the current parameter estimate is:
$$
\theta^{(m)}
$$
The main EM object is:
$$
Q(\theta\mid\theta^{(m)})
=
\mathbb{E}_{S_{1:T}\mid y_{1:T};\theta^{(m)}}
\left[
\log p(y_{1:T},S_{1:T}\mid\theta)
\right]
$$
This means:
> Take the complete-data log-likelihood and average it over the posterior distribution of the hidden states under the current parameters.

The posterior expectation of the state indicator is:
$$
\mathbb{E}[z_t(j)\mid y_{1:T};\theta^{(m)}]
=
\Pr(S_t=j\mid y_{1:T};\theta^{(m)})
=
\gamma_t^{(m)}(j)
$$
The posterior expectation of the transition indicator is:
$$
\mathbb{E}[z_t(i,j)\mid y_{1:T};\theta^{(m)}]
=
\Pr(S_{t-1}=i,S_t=j\mid y_{1:T};\theta^{(m)})
=
\xi_t^{(m)}(i,j)
$$
Therefore:
$$
Q(\theta\mid\theta^{(m)})
=
\sum_{j=1}^K \gamma_1^{(m)}(j)\log \pi_j
+
\sum_{t=2}^T
\sum_{i=1}^K
\sum_{j=1}^K
\xi_t^{(m)}(i,j)\log A_{ij}
+
\sum_{t=1}^T
\sum_{j=1}^K
\gamma_t^{(m)}(j)\log f_j(y_t)
$$
#### Intuition
EM does not assign each observation to exactly one regime.
Instead, it uses soft assignments.
Observation $y_t$ contributes to regime $j$ with weight:
$$
\gamma_t(j)
$$
Transition $i\to j$ at time $t$ contributes with weight:
$$
\xi_t(i,j)
$$
So EM is like maximum likelihood with hidden labels replaced by posterior probabilities.
### E-step
The E-step computes the posterior quantities needed for $Q$.
At iteration $m$, using current parameters $\theta^{(m)}$, compute:
$$
\gamma_t^{(m)}(j)
=
\Pr(S_t=j\mid y_{1:T};\theta^{(m)})
$$
and:
$$
\xi_t^{(m)}(i,j)
=
\Pr(S_{t-1}=i,S_t=j\mid y_{1:T};\theta^{(m)})
$$
In this model, the E-step consists of:
```

filter(obs, θ^(m))  
smooth(filter_result, θ^(m))  
pairwise(filter_result, smoother_result, θ^(m))

````
The outputs are:

| Step                 | Mathematical output | Purpose                 |                             |                                     |
| -------------------- | ------------------- | ----------------------- | --------------------------- | ----------------------------------- |
| forward filter       | $\alpha_{t          | t}$, $\alpha_{t         | t-1}$, $\ell(\theta^{(m)})$ | likelihood and inputs for smoothing |
| backward smoother    | $\gamma_t^{(m)}(j)$ | state occupancy weights |                             |                                     |
| pairwise calculation | $\xi_t^{(m)}(i,j)$  | transition weights      |                             |                                     |

#### Intuition
The E-step asks:
> Given the current model parameters, how likely is each hidden state and each hidden transition?

It does not change the parameters.
It only computes the posterior quantities needed to update them.
### M-step
The M-step treats the E-step posterior probabilities as fixed and updates the parameters by maximizing:
$$
Q(\theta\mid\theta^{(m)})
$$
So:
$$
\theta^{(m+1)}
=
\arg\max_{\theta} Q(\theta\mid\theta^{(m)})
$$
Because $Q$ separates into initial, transition, and emission terms, the updates can be written separately.
### Initial distribution update
The part of $Q$ involving $\pi$ is:
$$
Q_\pi
=
\sum_{j=1}^K \gamma_1^{(m)}(j)\log \pi_j
$$
with constraint:
$$
\sum_{j=1}^K \pi_j = 1
$$
The update is:
$$
\pi_j^{(m+1)}
=
\gamma_1^{(m)}(j)
$$
#### Intuition
The new initial probability of regime $j$ is the posterior probability that the first hidden state was regime $j$.
So if:
$$
\gamma_1^{(m)}(2)=0.8
$$
then:
$$
\pi_2^{(m+1)}=0.8
$$
### Transition matrix update
The part of $Q$ involving $A$ is:
$$
Q_A
=
\sum_{t=2}^T
\sum_{i=1}^K
\sum_{j=1}^K
\xi_t^{(m)}(i,j)\log A_{ij}
$$
Define expected transition counts:
$$
N_{ij}^{(m)}
=
\sum_{t=2}^T \xi_t^{(m)}(i,j)
$$
Then:
$$
Q_A
=
\sum_{i=1}^K
\sum_{j=1}^K
N_{ij}^{(m)}\log A_{ij}
$$
Each row of $A$ must satisfy:

$$
\sum_{j=1}^K A_{ij}=1
$$

The update is:
$$
A_{ij}^{(m+1)}
=
\frac{
N_{ij}^{(m)}
}{
\sum_{\ell=1}^K N_{i\ell}^{(m)}
}
$$
Equivalently, using:
$$
\sum_{\ell=1}^K \xi_t^{(m)}(i,\ell)
=
\gamma_{t-1}^{(m)}(i)
$$
we get:
$$
A_{ij}^{(m+1)}
=
\frac{
\sum_{t=2}^T \xi_t^{(m)}(i,j)
}{
\sum_{t=1}^{T-1}\gamma_t^{(m)}(i)
}
$$
#### Intuition
This is the soft-count version of estimating a Markov transition matrix.
If states were observed:
$$
A_{ij}
=
\frac{
\text{number of observed transitions } i\to j
}{
\text{number of observed transitions out of } i
}
$$
Since states are hidden:

$$
A_{ij}
=
\frac{
\text{expected number of transitions } i\to j
}{
\text{expected number of transitions out of } i
}
$$
### Gaussian mean update
The observation part of $Q$ is:
$$
Q_{\mathrm{obs}}
=
\sum_{t=1}^T
\sum_{j=1}^K
\gamma_t^{(m)}(j)\log f_j(y_t)
$$
For Gaussian emissions:
$$
y_t\mid S_t=j \sim \mathcal{N}(\mu_j,\sigma_j^2)
$$
Define the total posterior weight of regime $j$:
$$
W_j^{(m)}
=
\sum_{t=1}^T \gamma_t^{(m)}(j)
$$
The mean update is:
$$
\mu_j^{(m+1)}
=
\frac{
\sum_{t=1}^T \gamma_t^{(m)}(j)y_t
}{
W_j^{(m)}
}
$$
#### Intuition
This is a posterior-weighted average.
Observation $y_t$ contributes strongly to regime $j$ if:
$$
\gamma_t^{(m)}(j)\approx 1
$$
and weakly if:
$$
\gamma_t^{(m)}(j)\approx 0
$$
So the model estimates each regime mean using soft regime memberships.
### Gaussian variance update
The variance update is:
$$
(\sigma_j^2)^{(m+1)}
=
\frac{
\sum_{t=1}^T
\gamma_t^{(m)}(j)
\left(y_t-\mu_j^{(m+1)}\right)^2
}{
W_j^{(m)}
}
$$
#### Intuition
This is a posterior-weighted variance.
Observations that are strongly assigned to regime $j$ contribute strongly to that regime’s variance.
A high-variance regime will collect more dispersed observations.
A low-variance regime will collect observations closer to its mean.
### Why smoothed probabilities are used
The M-step uses:
$$
\gamma_t(j)
=
\Pr(S_t=j\mid y_{1:T})
$$
not filtered probabilities:
$$
\alpha_{t|t}(j)
=
\Pr(S_t=j\mid y_{1:t})
$$
because parameter estimation is an offline procedure using the full dataset.
Filtered probabilities only use observations up to $t$.
Smoothed probabilities use the entire sample.
Therefore, $\gamma_t(j)$ provides the most informative posterior weight for estimating the regime-specific parameters.
### Convergence criterion
At each iteration, the model evaluates the observed-data log-likelihood:
$$
\ell(\theta^{(m)})
=
\log p(y_{1:T}\mid\theta^{(m)})
$$
The forward filter computes:
$$
\ell(\theta^{(m)})
=
\sum_{t=1}^T \log c_t
$$
The algorithm stops when the improvement is sufficiently small:
$$
\left|
\ell(\theta^{(m+1)})
-
\ell(\theta^{(m)})
\right|
<
\varepsilon
$$
or when a maximum number of iterations is reached.
#### Intuition
If the log-likelihood barely changes between iterations, the fitted parameters are no longer improving meaningfully.
So the EM loop is considered converged.
### EM monotonicity
In exact EM, the observed-data log-likelihood is nondecreasing:
$$
\ell(\theta^{(m+1)})
\ge
\ell(\theta^{(m)})
$$
This means that each EM iteration should improve or maintain the likelihood.
In practice, tiny numerical decreases can occur because of floating-point arithmetic, numerical safeguards, or variance floors.
So implementations usually store the likelihood history and warn only if the decrease is larger than a numerical tolerance.
### Numerical and degeneracy guards
The implementation should handle degenerate cases.
#### Weight floor
The mean and variance updates divide by:
$$
W_j^{(m)}
=
\sum_{t=1}^T \gamma_t^{(m)}(j)
$$
If:
$$
W_j^{(m)} \approx 0
$$
then regime $j$ has almost no posterior mass.
The updates become unstable because they divide by a tiny number.
So the implementation may skip or freeze updates for regimes whose posterior weight is too small.
#### Variance floor
The Gaussian variance must satisfy:
$$
\sigma_j^2 > 0
$$
In EM, a regime may collapse onto very few observations and produce a variance close to zero.
To prevent this, the implementation enforces:

$$
\sigma_j^2 \ge \texttt{VAR\_FLOOR}
$$

#### Monotonicity tolerance
In theory:

$$
\ell(\theta^{(m+1)}) \ge \ell(\theta^{(m)})
$$
In floating-point arithmetic, small violations may happen.
So the implementation uses a tolerance before treating a likelihood decrease as meaningful.
### Full EM algorithm

```text
Input:
    observations y_1,...,y_T
    initial parameters θ^(0)
    tolerance ε
    maximum iterations M

For m = 0,1,...,M-1:

    E-step:
        Run forward filter under θ^(m)
            obtain α_{t|t}, α_{t|t-1}, log L(θ^(m))

        Run backward smoother
            obtain γ_t^(m)(j)

        Compute pairwise transition probabilities
            obtain ξ_t^(m)(i,j)
            obtain expected transitions N_ij^(m)

    M-step:
        π_j^(m+1) = γ_1^(m)(j)

        A_ij^(m+1)
            = [sum_{t=2}^T ξ_t^(m)(i,j)]
              / [sum_{t=1}^{T-1} γ_t^(m)(i)]

        μ_j^(m+1)
            = [sum_{t=1}^T γ_t^(m)(j)y_t]
              / [sum_{t=1}^T γ_t^(m)(j)]

        (σ_j²)^(m+1)
            = [sum_{t=1}^T γ_t^(m)(j)(y_t-μ_j^(m+1))²]
              / [sum_{t=1}^T γ_t^(m)(j)]

    Check convergence:
        stop if |log L(θ^(m+1)) - log L(θ^(m))| < ε

Output:
    fitted parameters θ_hat
    log-likelihood history
    convergence status
````

### Relationship to complete-data log-likelihood
The complete-data log-likelihood contains unknown indicators:
$$  
z_t(j)

\mathbf{1}{S_t=j}  
$$
and: $$  
z_t(i,j)

\mathbf{1}{S_{t-1}=i,S_t=j}  
$$
EM replaces them with their posterior expectations:
$$  
z_t(j)  
\leadsto  
\gamma_t(j)  
$$
and:
$$  
z_t(i,j)  
\leadsto  
\xi_t(i,j)  
$$
This is why the expected complete-data log-likelihood becomes:
$$  
Q(\theta\mid\theta^{(m)})

\sum_j \gamma_1^{(m)}(j)\log \pi_j  
+  
\sum_{t=2}^T\sum_i\sum_j  
\xi_t^{(m)}(i,j)\log A_{ij}  
+  
\sum_{t=1}^T\sum_j  
\gamma_t^{(m)}(j)\log f_j(y_t)  
$$
The first term updates $\pi$.
The second term updates $A$.
The third term updates the Gaussian emission parameters.

The hidden regimes make direct parameter estimation difficult.

## 3.9 Interpretation of fitted regimes

### What this section should do

This section explains how to interpret the fitted Markov switching model after estimation.

At this point, the model has produced fitted parameters:

$$
\hat{\theta}
=
(\hat{\pi},\hat{A},\hat{\mu}_1,\dots,\hat{\mu}_K,\hat{\sigma}_1^2,\dots,\hat{\sigma}_K^2)
$$

and posterior regime probabilities such as:

$$
\alpha_{t|t}(j)
=
\Pr(S_t=j\mid y_{1:t})
$$

and:

$$
\gamma_t(j)
=
\Pr(S_t=j\mid y_{1:T})
$$

The purpose of this section is to explain what these fitted quantities mean.

The main interpretation objects are:

| Object | Meaning |
|---|---|
| $\hat{\pi}$ | fitted initial regime probabilities |
| $\hat{A}$ | fitted transition and persistence structure |
| $\hat{\mu}_j$ | fitted regime-specific mean |
| $\hat{\sigma}_j^2$ | fitted regime-specific variance |
| $\alpha_{t|t}(j)$ | filtered / online posterior probability |
| $\gamma_t(j)$ | smoothed / retrospective posterior probability |

The central idea is:

> The regimes are not directly observed. They are inferred probabilistically from the fitted model.

Therefore, interpretation should be based on:

1. fitted parameters;
2. posterior probabilities;
3. diagnostic checks.

### What this section should not include

Keep this section focused on interpretation.

Do **not** rederive filtering, smoothing, pairwise probabilities, or EM.

Do **not** introduce new model variants such as Student-$t$ emissions, time-varying transitions, or semi-Markov models.

Do **not** discuss model selection criteria such as AIC or BIC.

Do **not** treat inferred regimes as observed truth.

Do **not** overinterpret numerical labels such as “regime 1” or “regime 2” before inspecting fitted parameters.

The purpose is only:

```text
fitted parameters + posterior probabilities + diagnostics
→ empirical interpretation of latent regimes
````

### Fitted parameter vector

After EM converges, the model returns:

$$  
\hat{\theta}

(\hat{\pi},\hat{A},\hat{\mu}_1,\dots,\hat{\mu}_K,\hat{\sigma}_1^2,\dots,\hat{\sigma}_K^2)  
$$

This fitted parameter vector summarizes the estimated latent regime structure.

Each part has a different interpretation:

|Parameter|Interpretation|
|---|---|
|$\hat{\pi}_j$|probability that the first hidden state was regime $j$|
|$\hat{A}_{ij}$|probability of moving from regime $i$ to regime $j$|
|$\hat{\mu}_j$|average level of observations in regime $j$|
|$\hat{\sigma}_j^2$|dispersion / volatility of observations in regime $j$|

The fitted model should be interpreted as a probabilistic description of latent regimes, not as a direct observation of them.

### Interpreting the initial distribution

The fitted initial distribution is:

$$  
\hat{\pi}

(\hat{\pi}_1,\dots,\hat{\pi}_K)  
$$

where:

$$  
\hat{\pi}_j

\widehat{\Pr}(S_1=j)  
$$

So $\hat{\pi}_j$ is the fitted probability that the time series begins in regime $j$.

#### Intuition

If:

$$  
\hat{\pi}_2 = 0.85  
$$

then the fitted model assigns high probability to the first hidden state being regime 2.

However, in long time series, $\hat{\pi}$ is usually less important for interpretation than:

$$  
\hat{A},\quad \hat{\mu}_j,\quad \hat{\sigma}_j^2,\quad \gamma_t(j)  
$$

because the initial distribution mainly affects the beginning of the sample.

### Interpreting the transition matrix

The fitted transition matrix is:

$$  
\hat{A}

(\hat{A}_{ij})_{i,j=1}^K  
$$
where:
$$  
\hat{A}_{ij}

\widehat{\Pr}(S_t=j\mid S_{t-1}=i)  
$$
This matrix describes the fitted switching behavior of the latent regimes.
The diagonal element:
$$  
\hat{A}_{jj}  
$$
is the fitted probability of staying in regime $j$ for one more time step.
The off-diagonal element:
$$  
\hat{A}_{ij},  
\qquad i\ne j  
$$
is the fitted probability of switching from regime $i$ to regime $j$.
#### Intuition
A large diagonal entry means that a regime is persistent.
For example:
$$  
\hat{A}_{22}=0.96  
$$
means that, conditional on being in regime 2, the fitted model assigns probability $0.96$ to remaining in regime 2 at the next time point.
A large off-diagonal entry means that the transition from one regime to another is relatively common.
For example:
$$  
\hat{A}_{12}=0.20  
$$
means that regime 1 has a fitted probability of $0.20$ of switching into regime 2 at the next time point.
### Expected regime duration
A useful inerpretation of the diagonal transition probabilities is expected regime duration.

Under the ordinary first-order Markov specification, the duration of regime $j$ is geometrically distributed.

The expected duration is:

$$  
\widehat{\mathbb{E}}[D_j]

\frac{1}{1-\hat{A}_{jj}}  
$$
#### Example

If:

$$  
\hat{A}_{jj}=0.95  
$$

then:

$$  
\hat{\mathbb{E}}[D_j]
\frac{1}{1-0.95}
$$

So the fitted model implies that regime $j$ lasts about 20 observations on average.

#### Intuition

Expected duration translates an abstract transition probability into an interpretable time-scale.

Instead of only saying:

$$  
\hat{A}_{jj}=0.95  
$$

we can say:

> The fitted model implies an average duration of approximately 20 time steps for regime $j$.

### Interpreting regime means

Each fitted regime has a mean:

$$  
\hat{\mu}_j  
$$

This is interpreted as the fitted conditional average of the observations in regime $j$:

$$  
\hat{\mu}_j  
\approx  
\mathbb{E}[y_t\mid S_t=j]  
$$

#### Intuition

A higher $\hat{\mu}_j$ means that observations assigned to regime $j$ tend to have higher values.

A lower $\hat{\mu}_j$ means that observations assigned to regime $j$ tend to have lower values.

The interpretation depends on what $y_t$ represents.

If $y_t$ is a return series, the fitted means may be small and less stable.

If $y_t$ is a volatility proxy, spread, transformed price movement, or other feature, then the fitted means may be easier to interpret as regime-specific levels.

### Interpreting regime variances

Each fitted regime has a variance:

$$  
\hat{\sigma}_j^2  
$$

This is interpreted as:

$$  
\hat{\sigma}_j^2  
\approx  
\operatorname{Var}(y_t\mid S_t=j)  
$$

In financial applications, the fitted variances are often among the most interpretable regime-specific parameters.

A regime with small fitted variance can be interpreted as a calm or stable regime.

A regime with large fitted variance can be interpreted as a volatile or turbulent regime.

|Fitted variance|Possible interpretation|
|---|---|
|small $\hat{\sigma}_j^2$|low-volatility / calm regime|
|large $\hat{\sigma}_j^2$|high-volatility / turbulent regime|

#### Important wording
The numerical labels of regimes are arbitrary.
Do not write:
> Regime 1 is the low-volatility regime.

unless you explicitly define regime 1 that way after fitting.
Better:
> The regime with the smallest fitted variance is interpreted as the low-volatility regime.

or:
> After ordering regimes by fitted variance, regime 1 corresponds to the low-volatility state.
### Label switching and arbitrary regime labels
The labels:
$$  
1,\dots,K  
$$
are not intrinsically meaningful.
If two regimes are relabeled and all corresponding parameters are relabeled consistently, the statistical model is equivalent.
For example, in a two-regime model, these two descriptions may represent the same fitted structure:

| Labeling A               | Labeling B               |
| ------------------------ | ------------------------ |
| regime 1 = low variance  | regime 1 = high variance |
| regime 2 = high variance | regime 2 = low variance  |
|                          |                          |

Therefore, interpretation should be based on fitted characteristics such as:
- smallest variance;
- largest variance;
- highest mean;
- longest expected duration;
- most frequent posterior occupancy.
Not on the raw numerical label alone.
### Interpreting posterior probabilities
The fitted parameters describe the model globally.
Posterior probabilities describe how regime membership changes over time.
The filtered posterior is:
$$  
\alpha_{t|t}(j)

\Pr(S_t=j\mid y_{1:t})  
$$
The smoothed posterior is:
$$  
\gamma_t(j)

\Pr(S_t=j\mid y_{1:T})  
$$
Filtered probabilities use data only up to time $t$.
Smoothed probabilities use the full sample.
#### Intuition
If:
$$  
\gamma_t(2)=0.92  
$$
then the model assigns high retrospective probability to regime 2 at time $t$.
This does not mean that regime 2 was directly observed.
It means:
> Given the fitted model and the full observed sequence, regime 2 is the most plausible latent state at time $t$.

### Filtered versus smoothed interpretation
Filtered probabilities:
$$  
\alpha_{t|t}(j)  
$$
are appropriate for online interpretation.

They answer:

> What did the model believe at time $t$ using information available up to time $t$?

Smoothed probabilities:
$$  
\gamma_t(j)  
$$
are appropriate for historical interpretation.
They answer:
> After seeing the whole sample, what does the model infer about the regime at time $t$?

#### Intuition
For historical regime plots, smoothed probabilities are usually preferable.
For online monitoring or real-time regime detection, filtered probabilities are the correct object.
### Hard regime labels
Sometimes posterior probabilities are converted into a single hard regime label:
$$  
\hat{S}_t

\arg\max_j \gamma_t(j)  
$$
This assigns each time point to the regime with the highest smoothed posterior probability.
But this hard label is only a summary.
It discards uncertainty.
#### Example
Both posterior vectors:
$$  
\gamma_t=(0.51,0.49)  
$$
and:
$$  
\gamma_t=(0.99,0.01)  
$$
produce:
$$  
\hat{S}_t=1  
$$
But the first case is uncertain, while the second case is highly confident.
So hard labels should not be treated as observed truth.
Better wording:
> The hard regime path is obtained by assigning each time point to the regime with the largest smoothed posterior probability.

Avoid:
> The true regime at time $t$ is $\hat{S}_t$.

### Posterior occupancy shares
A useful summary of regime usage is the posterior occupancy share:
$$  
\hat{o}_j
\frac{1}{T}  
\sum_{t=1}^T \gamma_t(j)  
$$
This measures the average posterior probability assigned to regime $j$ across the sample.
#### Intuition
If:
$$  
\hat{o}_2=0.35  
$$
then the model assigns about 35% of the sample, in posterior-weighted terms, to regime 2.
This is better than simply counting hard labels because it preserves uncertainty.
A regime with very small occupancy may be weakly identified or practically unused.
### Diagnostic checks for fitted regimes
Fitted regimes should be interpreted only after basic validity checks.
Important checks include:

| Diagnostic               | Why it matters                                                                    |
| ------------------------ | --------------------------------------------------------------------------------- |
| parameter validity       | $\hat{\pi}$ and rows of $\hat{A}$ must be valid probability distributions         |
| positive variances       | each $\hat{\sigma}_j^2$ must be positive                                          |
| non-degenerate variances | variance should not collapse near zero                                            |
| posterior normalization  | posterior probability vectors should sum to one                                   |
| meaningful occupancy     | regimes should not be nearly unused without explanation                           |
| reasonable persistence   | $\hat{A}_{jj}$ should not be suspiciously close to one without explanation        |
| likelihood path          | EM log-likelihood should be monotone or nearly monotone                           |
| multi-start stability    | different initializations should not produce radically different fitted solutions |

#### Intuition
A fitted regime may look interpretable but still be unreliable.
For example:
- a regime with almost zero occupancy may be unused;
- a regime with variance near zero may be degenerate;
- a regime with $\hat{A}_{jj}\approx 1$ may be nearly absorbing;
- different EM starts may produce different local optima.
So interpretation should combine parameter values, posterior probabilities, and diagnostics.

This section can briefly mention useful plots for empirical interpretation.
Useful figures include:

- observations $y_t$ with regime shading;
- fitted transition matrix heatmap;
- log-likelihood history from EM;
- posterior occupancy shares;
- expected regime durations.
These plots should support regime interpretation rather than introduce new methodology.

$$  
\hat{\theta}

(\hat{\pi},\hat{A},\hat{\mu}_1,\dots,\hat{\mu}_K,\hat{\sigma}_1^2,\dots,\hat{\sigma}_K^2)  
$$
The transition matrix $\hat{A}$ describes the fitted persistence and switching behavior of the latent Markov chain. In particular, $\hat{A}_{jj}$ is the probability of remaining in regime $j$, and the implied expected duration is:
$$  
\widehat{\mathbb{E}}[D_j]

\frac{1}{1-\hat{A}_{jj}}  
$$
The fitted Gaussian parameters describe the conditional distribution of the observations within each regime. The mean $\hat{\mu}_j$ represents the regime-specific average level of the observed variable, while $\hat{\sigma}_j^2$ represents the regime-specific variance. In financial applications, regimes are often interpreted primarily through their fitted variances: a low-variance regime may correspond to calm market conditions, while a high-variance regime may correspond to a turbulent period.
Regime interpretation should be based on posterior probabilities rather than only on hard labels. The filtered probabilities:
$$  
\alpha_{t|t}(j)=\Pr(S_t=j\mid y_{1:t})  
$$
describe online regime beliefs, while the smoothed probabilities:
$$  
\gamma_t(j)=\Pr(S_t=j\mid y_{1:T})  
$$
describe retrospective regime probabilities using the full sample. A hard classification:
$$  
\hat{S}_t=\arg\max_j \gamma_t(j)  
$$
may be used as a summary, but it should not be treated as an observed regime sequence. The posterior probabilities preserve uncertainty about regime membership.
Finally, the fitted regimes should be checked for numerical and statistical coherence. Important diagnostics include parameter validity, non-degenerate variances, normalized posterior probabilities, meaningful regime occupancy, reasonable expected durations, monotonicity of the EM likelihood path, and stability across multiple initializations.

### Key conceptual summary
The fitted parameter vector is:
$$  
\hat{\theta}

(\hat{\pi},\hat{A},\hat{\mu},\hat{\sigma}^2)  
$$

The transition matrix describes persistence and switching:
$$  
\hat{A}_{ij}

\widehat{\Pr}(S_t=j\mid S_{t-1}=i)  
$$
Expected duration is:

$$  
\widehat{\mathbb{E}}[D_j]

\frac{1}{1-\hat{A}_{jj}}  
$$
The emission parameters describe regime-specific behavior:
$$  
\hat{\mu}_j  
\approx  
\mathbb{E}[y_t\mid S_t=j]  
$$
$$  
\hat{\sigma}_j^2  
\approx  
\operatorname{Var}(y_t\mid S_t=j)  
$$
Posterior probabilities describe time-varying regime membership:
$$  
\alpha_{t|t}(j)

\Pr(S_t=j\mid y_{1:t})  
$$
$$  
\gamma_t(j)

\Pr(S_t=j\mid y_{1:T})  
$$
In one sentence:
> Interpreting fitted regimes means connecting estimated transition behavior, regime-specific Gaussian parameters, and posterior regime probabilities to empirically meaningful latent states, while remembering that regimes are inferred probabilistically rather than observed directly.
